# 易経変化ロジックDB 戦略ロードマップ

## ビジョン

**「実例に基づく易経診断」で、人生・ビジネスの転換期に具体的な洞察を提供する**

従来の抽象的な易経占いではなく、673件（目標1000件以上）の実例データに基づいた、科学的かつ実用的な診断ツールを提供する。

### データ品質管理

**品質基準**:
1. **信頼性（Credibility）**
   - S: 公式記録、学術研究、権威ある伝記
   - A: 大手メディア、著名な書籍
   - B: 一般メディア、信頼できるブログ
   - C: SNS、個人ブログ

2. **完全性（Completeness）**
   - 必須フィールド100%入力
   - story_summaryは100文字以上
   - logic_memoで易経的解釈を明記

3. **一貫性（Consistency）**
   - 八卦と変爻の整合性チェック
   - パターン分類の適切性
   - 類似事例との重複チェック

**品質保証プロセス**:
- 週次レビュー会（30件/回）
- 月次統計分析（偏り検出）
- 四半期監査（credibility再評価）

**推定精度**: 85%（主観的評価）

---

### 機械学習導入戦略

#### Phase 1: 教師あり学習（6ヶ月後〜）

**目的**: outcomeの予測精度向上

**アプローチ**:
```python
# 特徴量エンジニアリング
features = [
    # 既存フィールド
    "scale", "before_state", "trigger_type", "action_type",
    "before_hex", "trigger_hex", "action_hex", "after_hex",

    # 変爻情報
    "changing_lines_1", "changing_lines_2", "changing_lines_3",

    # 新規特徴量（エンジニアリング）
    "total_changes",  # 総変爻数
    "hex_pair_1_success_rate",  # before→triggerペアの歴史的成功率
    "hex_pair_2_success_rate",
    "hex_pair_3_success_rate",
    "pattern_type_success_rate",  # パターンタイプの成功率
    "credibility_rank_num",  # S=4, A=3, B=2, C=1
]

target = "outcome"  # Success/PartialSuccess/Mixed/Failure

# モデル候補
models = [
    "RandomForest",  # 解釈性◎、精度○
    "XGBoost",  # 精度◎、解釈性○
    "LightGBM",  # 高速、精度◎
]
```

**評価指標**:
- Accuracy: 全体正答率
- F1-Score (weighted): クラスバランス考慮
- AUC-ROC: 予測確率の品質

**目標精度**: 90%以上

#### Phase 2: 埋め込み学習（12ヶ月後〜）

**目的**: 事例間の意味的類似度を学習

**アプローチ**:
```python
# Sentence-BERTで story_summary を埋め込み
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')

# 各事例のストーリーをベクトル化
embeddings = model.encode([case.story_summary for case in all_cases])

# ユーザー入力も同様にベクトル化
user_embedding = model.encode(user_story)

# コサイン類似度でランキング
from sklearn.metrics.pairwise import cosine_similarity
similarities = cosine_similarity([user_embedding], embeddings)[0]
```

**期待効果**:
- キーワード一致以上の意味的類似性捕捉
- ユーザーが自由記述した状況にも対応可能

#### Phase 3: ハイブリッドモデル（18ヶ月後〜）

**アプローチ**:
```
最終スコア =
  0.4 × ルールベーススコア（現行アルゴリズム）
  + 0.3 × ML予測スコア（RandomForest等）
  + 0.3 × 埋め込み類似度スコア（SBERT）
```

**メリット**:
- 解釈性と精度の両立
- 各手法の長所を活用
- ロバスト性向上

### リスク1: データ収集の遅延
**対策**:
- 外部協力者の募集（クラウドソーシング）

**対策**:
- MVPの範囲を明確化（最小機能での公開）
- ノーコード/ローコードツールの活用検討
- 外部開発者の起用

### リスク4: 機械学習の精度不足
**対策**:
- データ量が十分になるまで導入延期
- ハイブリッドアプローチで段階的導入
