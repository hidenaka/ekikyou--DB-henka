# Claude Opus 用 作業指示

## 目的
`data/reference/iching_texts_ctext_legge.json`（Legge英訳＋漢文）を元に、
**わかりやすい日本語訳（HaQeiトーン）**を追加した分離データを作成する。

## 参照フォルダ（必読）
`docs/opus_handoff/refs/`
- `language_rulebook.md`（最重要）
- `killer_words_analysis.md`
- `voice_personality.md`
- `hook_patterns.md`
- `evidence_block.md`

## 入力 / 出力
- 入力: `data/reference/iching_texts_ctext_legge.json`
- 出力: `data/reference/iching_texts_ctext_legge_ja.json`
- **英訳データは変更しない**（入力はそのまま残す）

## 翻訳ルール（最重要）
- **基本トーン**: 寄り添い × 温かい言い切り
- **3層構造**: 肯定・安心 → 代弁・言語化 → 提案・選択肢  
  ※1ブロック1〜3文に収め、過剰な膨張は避ける
- **訳文の長さ**: 1〜3文、**40〜120文字目安**（長すぎる説明は避ける）
- **敬語/文体**: 丁寧語（です/ます調）。命令口調はNG。疑問形で終えない
- **ペルソナ寄り**:
  - ユウタ/ミサキ双方に届く**中立語彙**を優先（例: 決断/迷い/タイミング/本当の自分/内なる声）
  - キャリア/スピリチュアルに極端に寄せない
  - 追加の補足は**1文以内**に抑える
- **禁止語**（出力に使わない）:
  - 「易経」「卦」「爻」「変爻」「運勢」「占い」「宿命」「当たる/外れる」
- **言い換え**:
  - 「易経」→「3000年前の知恵」
  - 「卦」→「パターン」
  - 「爻」→「段階/フェーズ」
  - 「変爻」→「変化のポイント」
- **終止**: 断定はするが命令しない（「〜すべき」はNG）
  - 末尾は「〜です/〜ます/〜だと思います/〜という見方もあります」を基本

## 出力フォーマット
入力JSONの構造は維持し、各テキストブロックに `modern_ja` を追加する。

例（抜粋）:
```json
{
  "hexagrams": {
```

## 品質チェック
- 禁止語が混入していないか
- 1ブロックの文量が長すぎないか
- 直訳過多で読みづらくなっていないか
- 説明の追加で意味が逸れていないか

## 補足
- 「いつ動くべきか、待つべきか」は**自然に入る文脈のみ**使用する
- 事実の追加や脚色はしない（翻訳のみ）
