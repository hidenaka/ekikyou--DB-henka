## 1. 事例ID定義（Case Boundary Rules）

### 1.1 事例とは何か
**1事例 = 1つの明確な境界を持つイベント**

| 条件 | 例 |
|------|-----|
| 開始日が特定できる | 「2024年3月に発表」 |
| 終了日/現状が特定できる | 「2025年1月に完了」「進行中」 |
| 主体が単一 | 「トヨタ」「孫正義」 |
| 変化の内容が1つ | 「EV転換」「事業売却」「創業」 |

### 1.2 分割が必要なケース
以下は別事例として分割:
- 同一企業でも異なる時期の異なる変化
- 同一プロジェクトでもフェーズが明確に異なる
- 経営者交代前後

### 1.3 統合が必要なケース
以下は1事例に統合:
- 同一イベントの複数報道
- 連続する施策群（1つの戦略の実行）

## 2. 重複排除基準（Duplicate Elimination）

### 2.1 重複の定義
```
重複 = (target_name, period) が既存DBに存在
```

### 2.2 チェック手順（必須）
```bash
# 収集前に必ず実行
python3 -c "
import json
existing = set()
with open('data/raw/cases.jsonl') as f:
    for line in f:
        case = json.loads(line)
        key = (case.get('target_name',''), case.get('period',''))
        existing.add(key)
# 新規ケースのチェック
new_key = ('企業名', '2024-2025')
print('重複' if new_key in existing else '新規')
"
```

### 2.3 目標
- **重複率 0%** を目指す（15/40は不合格）
- 類似事例（同一企業・異期間）は許容

## 3. 爻判定基準（Yao Assignment Rules）

### 3.1 爻の意味（6段階）

| 爻 | 位置 | 意味 | キーワード |
|----|------|------|------------|
| 1爻 | 最下 | 始まり・萌芽・基盤 | 創業、着手、準備 |
| 2爻 | 下位 | 発展・内部成長 | 展開、成長、基礎固め |
| 3爻 | 中下 | 転換点・岐路 | 転換、選択、危機 |
| 4爻 | 中上 | 外部展開・進出 | 拡大、進出、挑戦 |
| 5爻 | 上位 | 成熟・頂点 | 成功、完成、統治 |
| 6爻 | 最上 | 終末・極限・変質 | 終焉、過剰、新展開 |

### 3.2 爻判定の手順

1. **時間軸で判定**: イベントはサイクルのどの位置か？
2. **規模で判定**: 内部完結(1-2)か外部展開(4-5)か？
3. **結果で判定**: 安定継続(2,5)か転換点(3,6)か？

### 3.3 爻判定の例

| 事例 | 爻 | 理由 |
|------|-----|------|
| 創業時の苦闘 | 1爻 | 始まりの段階 |
| 国内シェア拡大 | 2爻 | 内部での発展 |
| IPOか事業売却かの選択 | 3爻 | 岐路・転換点 |
| 海外進出決定 | 4爻 | 外部への展開 |
| 業界トップ達成 | 5爻 | 頂点・成熟 |
| 過剰拡大による失敗 | 6爻 | 極限・終末 |

## 4. logic_memoテンプレート（必須5要素）

### 4.1 構造
```
logic_memo = [局面] + [関係性] + [選択] + [結果] + [反証爻]
```

### 4.2 要素の定義

| 要素 | 説明 | 例 |
|------|------|-----|
| **局面** | before_hexの状態を具体的に | 「巨大だが動きの遅い組織（坤）」 |
| **関係性** | trigger_hexとの関係 | 「巨額赤字という落とし穴（坎）に転落」 |
| **選択** | action_hexの行動 | 「荒療治（震）で上場子会社を整理」 |
| **結果** | after_hexの状態 | 「ITとインフラ融合でリーダーシップ（乾）を回復」 |
| **反証爻** | この爻でない理由 | 「創業期ではない(≠1爻)、極限でもない(≠6爻)」 |

### 4.3 良い例
```
「巨大だが動きの遅い組織（坤）が、巨額赤字という落とし穴（坎）に転落。
川村・中西体制による雷のような荒療治（震）で、上場子会社の整理や
ハードウェアからの脱却を断行。ITとインフラの融合で再びリーダーシップ（乾）
を取り戻した。【4爻判定理由: 外部展開フェーズ、創業期でも極限でもない】」
```

### 4.4 悪い例（NG）
```
「ピボットに成功した。」  # 短すぎる、要素不足
「良い変化が起きた。」    # 八卦との対応なし
```

## 5. ソース品質基準（Source Quality）

### 5.1 一次情報（優先）

| ランク | ソース種別 | 例 |
|--------|------------|-----|
| **S** | 公式発表・IR | 有価証券報告書、プレスリリース |
| **A** | 判決文・行政文書 | 裁判所判決、公正取引委員会発表 |
| **A** | 経済専門紙 | 日経新聞、WSJ、Bloomberg |

### 5.2 二次情報（条件付き可）

| ランク | ソース種別 | 条件 |
|--------|------------|------|
| **B** | 一般ニュース | 複数ソースで確認 |
| **B** | 書籍・論文 | 著者・出版社が明確 |
| **C** | Wikipedia | 出典が明記されている場合のみ |

### 5.3 不可ソース
- 個人ブログ（出典なし）
- SNS投稿のみ
- 匿名掲示板
