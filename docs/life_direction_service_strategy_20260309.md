# 人生の方向診断 事業方針メモ

**作成日**: 2026-03-09  
**目的**: このプロジェクトの事業ポジションについて、今後のエージェント作業で解釈がぶれないようにするための方針整理。

## 結論

このDBの主戦場は、仕事紹介や転職斡旋ではない。  
**「順調に見える人生の中にある違和感」を整理し、現在地・未来の分岐・次の一歩を示す人生の方向診断サービス**として育てる。

仕事やキャリアは重要な出口の1つだが、本質ではない。  
本質は、**まだテーマが言語化できていない不安や停滞感を、意思決定可能な形に変えること**にある。

## 想定顧客

最重要の顧客像は以下。

- 外から見ると大きな問題はなく、むしろ順調に見える
- しかし本人の内側には「このままでいいのか」がある
- 転職・独立・移住など、結論がまだ決まっていない
- 何かを変えたい気はあるが、何から動けばいいか分からない
- 一人で考えても整理しきれず、視点と順番が欲しい

一文で言うと、

**「順調に見える人生の中で違和感を抱え、このままでいいのかを整理したい人」**

## 非ターゲット

以下は主対象にしない。

- 仕事そのものの紹介や斡旋を期待する人
- 具体的な求人・案件・転職先の提案を求める人
- 心身の不調が強く、まず医療・心理支援が必要な人
- 娯楽としてだけ占いを楽しみたい人
- 自分で動く意思はなく、正解だけを受け取りたい人

## サービスの役割

このサービスは「答えを言い当てるもの」ではない。  
役割は以下の4つに絞る。

1. 今の違和感を現在地として言語化する
2. このまま進んだ場合に起こりやすい未来の流れを見せる
3. 別ルートに進む場合の選択肢を2〜3本提示する
4. 次の30日・90日で試すべき小さな行動を示す

つまり、**不安を消すサービスではなく、不安を構造化して動ける状態を作るサービス**である。

## 提供価値

ユーザーが金を払う理由は、「不安があるから」だけでは弱い。  
支払理由は、**その不安を整理し、次の方向を決めたいから**である。

したがって価値提案は次のように置く。

- 漠然とした不安を整理できる
- 今の自分の局面が分かる
- 今の延長線の未来を俯瞰できる
- 別の可能性を比較できる
- 今日から試す行動が決まる

## ポジショニング

避けるべき表現:

- 天職を当てる
- 人生の正解を教える
- 不安を治す
- 転職先を示す

採用すべき表現:

- 現在地を整理する
- 未来の分岐を見る
- 次の一歩を決める
- 違和感を言語化する
- 変化の地図を手に入れる

定義としては、

**占い寄りの演出を使えても、本体は「実例データに基づく未来分岐の診断」**である。

## 商品コンセプト

最初の商品は、人生全体を大上段で断定するものではなく、次の形がよい。

**「このまま続ける未来 / 変える未来 / 理想に寄せる未来」を整理する現在地診断レポート**  

標準出力の骨子:

- 現在地の分析
- 今のまま進んだ場合の流れ
- 別ルート候補 2〜3本
- 各ルートの向き・リスク・必要行動
- 今日やること 1つ
- 30日 / 90日の実験プラン

## 命名・コピー方針

現時点の第一候補:

- サービス名: **人生の現在地診断**
- サブタイトル: **このままでいいのかを整理する、未来分岐のデータ診断**

一言説明:

**順調に見える毎日の中にある違和感を、現在地・未来の分岐・次の一歩に整理する診断サービス**

LP冒頭コピー案:

**このままでいいのか。  
その違和感を、曖昧なままにしない。**

補助コピー案:

- 人生が壊れている人のためではなく、壊れてはいないが、このままの延長に違和感がある人のための診断
- 答えを押しつけるのではなく、次の一歩を見つけるための整理
- すぐに人生を変えるのでなく、まず進む方向を見える化する

## このDBとの整合性

この方針はデータ構造とも整合する。

- 実例DBは、卦辞の解釈だけでなく、**状態 → 行動 → 結果** の遷移を持つ
- サンプルレポートは既に **現在地分析 / 推奨ルート / 今日やること** の形式を持つ
- API実装には、段階診断や90日行動計画の枠組みが既にある

2026-03-09時点のローカル集計では、`data/raw/cases.jsonl` に 11,336 件、うち `individual` は 3,089 件ある。  
したがって、仕事紹介機能がなくても、**「人生の分岐に対する診断」**として十分に成立しうる。

## 将来の設計原則

今後、機能・コピー・UI・商品設計を行う際は以下を守る。

1. キャリア支援に寄せすぎない
2. 仕事紹介を前提にしない
3. ユーザーの違和感をまず言語化する
4. 大きな決断の断定より、小さな次の一歩を重視する
5. 神秘性より、整理・比較・行動設計を前面に出す

## 次の実務ステップ

- この方針に沿ってLP文言を設計する
- 質問フローを「問題解決」型ではなく「違和感の言語化」型に寄せる
- 初回商品をレポート形式で定義する
- ココナラやLINE向けに短い説明文へ落とす

## Future Agent向けメモ

今後このプロジェクトの事業提案を行う場合、初期案として以下を前提にすること。

- 事業の中心は「人生の方向診断」
- 顧客は「順調だが違和感がある人」
- 提供価値は「現在地・未来分岐・次の一歩の整理」
- 転職斡旋や求人提案はコア価値ではない
