**前提**: Phase 0-3 完了（ANALOGOUS判定）、LLM Debate結論（診断エンジン不成立、シナリオ・エクスプローラー推奨）

## 0. 設計原則

### 絶対に守ること
1. **「予測」という語を使わない**。出力は「変化の構造的読み解き」と「参照分布」である
2. **確定的遷移ルールを示さない**。「卦Aの次は卦Bになる」は易経に存在しない
3. **事例件数(n)を必ず併記する**。小標本の高確率を断定に使わない
4. **易経の用語を神秘化しない**。全ての概念を構造的な意味に落とす
5. **入力の不確実性を隠さない**。マッピング確信度を明示する

### 使えるメカニズム（原典に存在するもの）
| メカニズム | 何を示すか | 出力での使い方 |
|-----------|-----------|---------------|
| 変爻 | 極まったものは反転する | 之卦の導出根拠 |
| 六爻フェーズ | 潜伏→出現→危機→選択→最盛→過剰 | 現在地の可視化 |
| 互卦 | 2-5爻の内的構造 | 表面の下にある本質的力学 |
| 錯卦 | 全爻反転 | 意識していない対極 |
| 綜卦 | 上下反転 | 視点の転換 |
| 窮則變 | 窮まれば変ず | 変化の必然性の根拠 |
| 一陰一陽之謂道 | 陰陽は循環する | 反転・サイクルの根拠 |

### 使ってはいけないもの
- 64x64の遷移行列に基づく「次の卦はXです」
- 特定の未来を断定する文言
- 条件付き確率表の数値を「予測精度」として提示

## 1. フィードバック構造（全体設計）

### 1回のフィードバックの構成要素と提示順序

```
┌─────────────────────────────────────────────┐
│  LAYER 1: 現在地の読み解き（本卦 + 動爻）     │  ← 必ず表示
│  ─────────────────────────────────────────── │
│  LAYER 2: 変化が向かう構造（之卦）             │  ← 必ず表示
│  ─────────────────────────────────────────── │
│  LAYER 3: 見えていない力学（互卦 + 錯卦 + 綜卦）│  ← 折りたたみ可
│  ─────────────────────────────────────────── │
│  LAYER 4: 過去事例から見える分布               │  ← 折りたたみ可
│  ─────────────────────────────────────────── │
│  LAYER 5: 問いかけ（内省を促す1問）             │  ← 必ず表示
└─────────────────────────────────────────────┘
```

**設計根拠**: 情報過多を防ぐため、LAYER 1-2-5は常時表示、LAYER 3-4はユーザーが開きたいときだけ展開する。最も重要なのは「今どこにいるか」（L1）と「変化はどの方向に向かうか」（L2）と「自分への問い」（L5）。

## 2. 各レイヤーの詳細設計

### LAYER 1: 現在地の読み解き（本卦 + 動爻）

#### 2.1.1 構成要素

```yaml
layer1:
  hexagram_visual:       # 卦の爻図（動爻をハイライト）
  hexagram_name:         # 「火天大有」など
  hexagram_number:       # 14 など
  trigram_composition:    # 上卦: 離（火）、下卦: 乾（天）
  judgment_modern_ja:     # 卦辞の現代日本語訳（iching_texts_ctext_legge_ja.jsonから）
  current_phase:         # 六段階のどこにいるか
  phase_visualization:   # フェーズバーの視覚化
  yao_text_modern_ja:    # 動爻の爻辞の現代日本語訳
  yao_xiang_modern_ja:   # 動爻の象伝の現代日本語訳
  mapping_confidence:    # マッピング確信度（0.0-1.0）
```

#### 2.1.2 六段階フェーズの可視化

動爻の位置をフェーズバーで表示する。

```
フェーズモデル:
  初爻  二爻  三爻  四爻  五爻  上爻
  潜伏  出現  転換  選択  最盛  極限
   ○    ○    ○    ●    ○    ○    ← 四爻が動爻の例

  [ ░░░░░░░░░░░████░░░░░░ ]
    潜伏  出現  転換 ▲選択 最盛  極限
                    ここ
```

テキスト形式での表現:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [潜伏] → [出現] → [転換] → [★選択] → [最盛] → [極限]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  あなたは今、6段階のうち第4段階「選択」にいます。
  新しいステージに入ったが、まだ足場が固まっていない時期です。
```

#### 2.1.3 フェーズごとの本質的メッセージ（line_positions.jsonから拡張）

| 爻位 | フェーズ名 | 本質 | キーワード |
|------|-----------|------|-----------|
| 初爻 | 潜伏 | まだ表に出る時ではない。力を蓄える | 準備、内省、観察 |
| 二爻 | 出現 | 内側で力をつけ、基礎を固めている | 基盤、着実、中庸 |
| 三爻 | 転換 | 内から外への境界。危険だが飛躍の入口 | 転換点、試練、慎重 |
| 四爻 | 選択 | 新しいステージの入口。不安定だが可能性 | 決断、適応、謙虚 |
| 五爻 | 最盛 | 最も力を発揮できる位置 | 中心、統率、責任 |
| 上爻 | 極限 | これ以上は進めない。次のサイクルへ | 過剰、転落の兆し、手放す |

#### 2.1.4 卦辞・爻辞のソース

**最優先**: `data/reference/iching_texts_ctext_legge_ja.json` の `modern_ja` フィールド
- `hexagrams[N].judgment.modern_ja` → 卦辞
- `hexagrams[N].lines[M].modern_ja` → 爻辞
- `hexagrams[N].lines[M].xiang.modern_ja` → 小象伝

**フォールバック**: `data/diagnostic/yao_384.json` → 384爻の解釈データ

### LAYER 2: 変化が向かう構造（之卦）

#### 2.2.1 構成要素

```yaml
layer2:
  resulting_hexagram_visual:   # 之卦の爻図
  resulting_hexagram_name:     # 之卦の名前
  resulting_hexagram_number:   # 之卦の番号
  transformation_explanation:  # 「第N爻が陽→陰に変化することで...」
  resulting_judgment_modern_ja: # 之卦の卦辞（現代日本語）
  structural_reading:          # 構造的読み解き（下記）
  transition_arrow:            # 本卦→之卦の変化矢印
```

#### 2.2.2 構造的読み解きの生成ルール

之卦は「変化が向かう構造」であり、「確定した未来」ではない。以下のテンプレートで生成する。

**テンプレート**:
```
[本卦名]の第[N]爻が変化すると、[之卦名]の構造が現れます。

[本卦]は[本卦の卦辞要約]という状態ですが、
第[N]爻の[陽/陰]が極まって[陰/陽]に転じることで、
[之卦]の[之卦の卦辞要約]という方向に構造が移ります。

これは「窮まれば変ず」（繋辞伝）の原理です。
今の状態が極まったとき、変化は[之卦が示す方向]に向かう力学を持っています。
```

**重要**: 「〜になります」「〜になるでしょう」は禁止。代わりに以下を使う:
- 「〜という方向に構造が移ります」
- 「〜に向かう力学を持っています」
- 「〜という構造が現れます」

#### 2.2.3 変爻の意味の解説

変爻が何を意味するかを、六段階フェーズモデルに基づいて解説する。

| 変爻位置 | 変化の性質 |
|---------|-----------|
| 初爻が変 | 始まりの性質が変わる。根本的な方向転換 |
| 二爻が変 | 基盤の在り方が変わる。内部構造の転換 |
| 三爻が変 | 転換点での判断が変わる。外への出方が変わる |
| 四爻が変 | 選択の質が変わる。新環境での立ち位置が変わる |
| 五爻が変 | 中心の在り方が変わる。リーダーシップの転換 |
| 上爻が変 | 極限の反転。サイクルの終わりと始まり |

### LAYER 3: 見えていない力学（互卦・錯卦・綜卦）

#### 2.3.1 構成要素

```yaml
layer3:
  nuclear:                      # 互卦
    hexagram_name: str
    hexagram_number: int
    reading: str                # 「表面の下にある力学」
    judgment_modern_ja: str

  complementary:                # 錯卦
    hexagram_name: str
    hexagram_number: int
    reading: str                # 「意識していない対極」
    judgment_modern_ja: str

  inverted:                     # 綜卦
    hexagram_name: str
    hexagram_number: int
    reading: str                # 「立場を変えた視点」
    judgment_modern_ja: str
```

#### 2.3.2 各変換の読み解きテンプレート

**互卦（内的本質）**:
```
表面に見えている[本卦名]の内側には、[互卦名]の力学が働いています。
[互卦の卦辞要約]
この内的構造が、今の状況を根底で形作っています。
```

**錯卦（対極の構造）**:
```
[本卦名]の全ての要素を反転させると、[錯卦名]が現れます。
これは今の状況の「影」、意識から外れている側面です。
[錯卦の卦辞要約]
この対極を認識することで、見えていなかったものが見えるかもしれません。
```

**綜卦（視点の転換）**:
```
[本卦名]を上下逆さまにすると、[綜卦名]になります。
同じ状況を、相手の立場・逆の視点から見た姿です。
[綜卦の卦辞要約]
視点を180度変えたとき、この構造が浮かび上がります。
```

#### 2.3.3 折りたたみの理由

互卦・錯卦・綜卦は「深掘りしたい人」向けの情報。初見で全てを提示すると情報過多になる。UI/CLI上では折りたたみ、API出力では全て含めるが `depth: "extended"` タグを付ける。

### LAYER 4: 過去事例から見える分布

#### 2.4.1 構成要素

```yaml
layer4:
  conditional_distribution:     # 条件付き確率分布
    condition:                  # 「停滞・閉塞 × 刷新・破壊」など
    total_n: int                # 該当事例数
    distribution:               # after_stateの分布
      - state: str
        count: int
        percentage: float
    caveat: str                 # 注意書き（下記）

  similar_cases:                # 類似事例
    - target_name: str
      story_summary: str
      before_state: str
      action_type: str
      after_state: str
      outcome: str
      similarity_basis: str     # 何に基づいて類似と判断したか

  pattern_distribution:         # パターン分布（オプション）
    total_n: int
    patterns:
      - pattern_type: str
        count: int
        percentage: float
```

#### 2.4.2 条件付き確率分布の提示ルール

**必須表示要素**:
1. 条件（before_state × action_type の組み合わせ）
2. 該当事例数 n
3. 上位3-5件の結果分布（パーセンテージ + 件数）
4. 注意書き

**注意書きテンプレート（n の値に応じて変化）**:

| n の範囲 | 注意書き |
|---------|---------|
| n >= 100 | 「同様の状況で[action]をとった{n}件の事例では、以下の分布が見られました。」 |
| 30 <= n < 100 | 「参考事例が{n}件あります。傾向の参考程度にご覧ください。」 |
| 10 <= n < 30 | 「該当事例は{n}件と少数です。分布は参考情報としてご覧ください。」 |
| n < 10 | 「該当事例は{n}件のみです。統計的な傾向を読み取ることは困難です。」 |

**表示形式**:
```
┌──────────────────────────────────────────┐
│  同様の状況 × 行動 の過去事例（442件）     │
│  条件: 停滞・閉塞 × 刷新・破壊            │
├──────────────────────────────────────────┤
│  持続成長・大成功  ███████████░░░░ 26.9%  │
│  変質・新生        ████████░░░░░░ 21.0%  │
│  安定成長・成功    █████░░░░░░░░░ 14.3%  │
│  崩壊・消滅        █████░░░░░░░░░ 13.3%  │
│  その他            ████░░░░░░░░░░ 24.4%  │
├──────────────────────────────────────────┤
│  ※ 過去事例の分布であり、                  │
│    あなたの結果を予測するものではありません │
└──────────────────────────────────────────┘
```

#### 2.4.3 類似事例の選出基準

優先順位（上位ほど優先）:
1. **本卦 + 動爻が一致** → 最も構造的に類似
2. **本卦が一致（動爻は異なる）** → 同じ状態構造
3. **before_state + action_type が一致** → 同じ状況×行動パターン
4. **before_hex + action_hex が一致** → 同じ八卦タグの組み合わせ

提示件数: **最大3件**。多すぎると焦点がぼやける。

各事例の提示形式:
```
  ■ [対象名]（[period]）
    状況: [before_state] → 行動: [action_type] → 結果: [after_state]
    概要: [story_summary（100文字以内に切り詰め）]
    ──
    ※ 類似の根拠: [本卦一致 / before_state+action_type一致 / etc.]
```

#### 2.4.4 「予測」と「参照分布」の境界線

| やること（参照分布） | やらないこと（予測） |
|---------------------|---------------------|
| 「過去{n}件ではこの分布でした」 | 「あなたは{X}%の確率で成功します」 |
| 「同じ行動をとった事例の結果はこう分かれました」 | 「この行動をとれば成功します」 |
| 「類似事例にはこのようなものがあります」 | 「あなたもこうなるでしょう」 |
| 分布の棒グラフ + 件数 | 単一の確率値をハイライト |
| 「参考情報」と明記 | 「診断結果」「予測結果」と称する |

### LAYER 5: 問いかけ（内省を促す1問）

#### 2.5.1 設計意図

フィードバックの最後は情報ではなく「問い」で終わる。ユーザーが受動的に情報を受け取るだけでなく、自分の状況を能動的に考えるための起点を作る。

#### 2.5.2 問いの生成ルール

動爻の位置に基づいて問いを生成する。

| 動爻 | 問いのテーマ | 例 |
|------|-------------|-----|
| 初爻 | 潜伏の意味 | 「今、表に出ないことで蓄えているものは何ですか？」 |
| 二爻 | 基盤の確認 | 「今固めている基盤の中で、最も確かなものは何ですか？」 |
| 三爻 | 転換の決意 | 「内側に留まるリスクと、外に出るリスク、どちらが大きいですか？」 |
| 四爻 | 選択の質 | 「新しいステージで、最初に手放すべきものは何ですか？」 |
| 五爻 | 責任の本質 | 「あなたが今、最も影響を与えている相手は誰ですか？」 |
| 上爻 | 手放し | 「ここまで来て、まだ握りしめているものは何ですか？」 |

さらに、之卦の性質を加味して問いを調整する。

                                │
                ┌───────────────┼───────────────┐
                ▼               ▼               ▼

                                │
                                ▼
                    [E. 出力（5レイヤー構造）]

## 6. 「予測」と「変化の本質の提示」の境界線

### 6.1 原則的区分

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   予測 (Prediction)          変化の本質の提示 (Structural   │
│   ── やらない                Reading) ── やる               │
│                                                             │
│   「あなたはXになる」        「この構造にはXに向かう        │
│                              力学がある」                    │
│                                                             │
│   「X%の確率で成功する」     「過去N件ではこの分布          │
│                              だった」                       │
│                                                             │
│   「このまま行けばY」        「第N爻が極まれば              │
│                              Y的な構造に転じる」            │
│                                                             │
│   未来の断定                 構造の読み解き                  │
│   単一の結論                 複数の可能性の提示              │
│   因果の主張                 相関と分布の提示                │
│   「あなたの運命」           「変化の力学」                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 文言のチェックリスト

出力テキストを生成する際、以下の禁止語・推奨語をチェックする。

**禁止語**:
- 「〜になるでしょう」「〜になります」（断定的未来形）
- 「予測」「予言」「運命」「運勢」「占い」
- 「〜%の確率で〜です」（個人の確率として提示）
- 「必ず」「確実に」「間違いなく」

**推奨語**:
- 「〜という構造が現れます」
- 「〜に向かう力学を持っています」
- 「過去の事例では〜という分布が見られました」
- 「〜という可能性が構造的に示唆されています」
- 「変化の方向性として〜が読み取れます」

### 6.3 哲学的根拠

易経の「変」の思想は、確定的未来を示すものではない。

- **窮則變**: 窮まれば変ず → 変化は「極まった構造」から生じる。構造が変化を生むのであって、時間が未来を決めるのではない
- **一陰一陽之謂道**: 陰と陽は常に循環する → 今の状態は永続しない。これは「いつ変わるか」の予測ではなく「変化の必然性」の指摘
- **生生之謂易**: 生じ続けることが変化の本質 → 固定的な未来像は易の思想と矛盾する

この3つの命題から導かれるのは:
「今の構造がどの方向に変化しうるか」は語れる。「いつ、確実にどうなるか」は語れない。

## 8. 今後の拡張余地

1. **ドメイン別分布**: `scale` (company/individual/country)で条件付き分布を分離する → データリーク防止
2. **信頼度による重み付け**: `trust_level = verified` の事例のみで分布を計算するオプション
3. **十二消息卦の季節性**: before_hexの八卦を十二消息卦と対応させ、「サイクルのどこにいるか」をLAYER 1に追加
4. **序卦伝の因果連鎖**: 本卦の前後の卦（序卦伝の順序）から「この卦に至った経緯」と「この卦の次に来る構造」を参照
5. **事例の事前/事後分離**: 新規事例を「事前登録」→「事後追跡」で収集し、予測精度の校正を組み込む（LLM Debateで指摘された評価設計）
