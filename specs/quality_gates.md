# Quality Gates — フェーズゲート定義

## L1: 不可侵制約（構造的に破れない壁）

### データ保護
- `data/raw/cases.jsonl` は全エージェントに対してRead Only
- 分析結果は `analysis/` ディレクトリにのみ出力
- 既存の `scripts/`, `harness/` を改変しない

### フェーズ順序
- Phase 2BはPhase 2A完了後にのみ開始（MCA結果がクラスタリングの入力）
- Phase 3はPhase 1 AND Phase 2完了後にのみ開始
- Phase 4（予測モデル）はPhase 3で同型性確認後にのみ開始

### 帰無仮説の方向
- 「同型性がある」を証明するのではなく「同型性がない」を帰無仮説とする
- 「6次元に収束する」を前提としない

### レビュー義務
- 各Phase完了後、quality-reviewerによるレビューを必須とする
- FAIL判定の場合、該当Phaseを再実行（最大2回まで）
- 2回FAILの場合、オーケストレーターが介入して方針を再検討

## L2: 参照パターン（8割はこれでカバー）

### エージェント配置パターン

```
Phase 1: state-space-modeler (単独)
         ↓ [quality-reviewer でレビュー]
Phase 2A: change-analyzer (単独)
         ↓ [quality-reviewer でレビュー]
Phase 2B: change-analyzer (単独、Phase 2Aの続き)
         ↓ [quality-reviewer でレビュー]
Phase 3: isomorphism-checker (単独)
         ↓ [quality-reviewer でレビュー]
```

### Phase 1 と Phase 2A の並行実行
- 相互依存がないため並行実行可能
- それぞれ独立したディレクトリに出力
- 完了後、Phase 2B → Phase 3 と順次進行

### コンテキスト管理
- 各エージェントはフレッシュコンテキストで起動（バイアス排除）
- 「記憶は消す、記録は残す」: 成果物はファイルに永続化
- quality-reviewerは毎回新しいコンテキストで起動（ポジティビティバイアス対策）

## L3: 状況判断（オーケストレーター裁量）

### 介入条件
- Phase完了が予想時間の2倍を超えた場合
- エージェントがNOT Allowedの行動を取った場合
- quality-reviewerが2回FAILを出した場合
- 予期しない結果（例: 次元数が2以下、または20以上）が出た場合

### 判断項目
- スコープの変更（分析対象の変数を増減）
- エージェントの再起動（フレッシュコンテキスト）
- Phase間の順序変更（正当な理由がある場合のみ）
- プロジェクト方針の再検討

## 成功KPI

| Phase | KPI |
|-------|-----|
| Phase 1 | 64卦グラフが数学的に正しい。グラフ性質が計算済み。可視化あり |
| Phase 2A | MCAで次元数が決定済み。各次元の解釈が明示。スクリープロットあり |
| Phase 2B | 遷移行列が構築済み。有意な遷移パターンが抽出済み。クラスタリング完了 |
| Phase 3 | 同型性の有無について統計的根拠のある結論。p値と効果量を報告 |
| 全体 | 前回の「根拠がない」問題が解消（肯定的でも否定的でも） |
