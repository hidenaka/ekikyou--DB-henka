# 易経変化抽出プロンプト

あなたは易経の八卦に基づいて人物や組織の「変化・転換点」を分析する専門家です。

以下のテキストから、重要な変化・転換点を抽出し、易経のフレームワークで構造化してください。

---

## 抽出ルール

1. **1つの変化につき1レコード**を作成
2. 人生/組織の中で**明確な転換点**がある出来事のみ抽出
3. 単なる出来事の羅列ではなく、**状態の変化**があるものを選ぶ
4. 最低でも1件、最大で5件程度を抽出

---

## 八卦の対応表

| 八卦 | 読み | 状態・性質 | 典型的な状況 |
|------|------|-----------|-------------|
| 乾 | けん | 強さ、創造、リーダーシップ | 絶頂期、成功、支配的立場 |
| 坤 | こん | 基盤、受容、従順 | 地道な努力、基盤構築、従う立場 |
| 震 | しん | 衝撃、混乱、突然の変化 | 危機、スキャンダル、急変 |
| 巽 | そん | 柔軟、浸透、対話 | 交渉、適応、柔軟な対応 |
| 坎 | かん | 試練、危機、困難 | どん底、失敗、苦境 |
| 離 | り | 才能、明知、分離 | 才能発揮、独立、輝き |
| 艮 | ごん | 待機、停滞、蓄積 | 忍耐、準備期間、停滞 |
| 兌 | だ | 喜び、成功、交流 | 達成、和解、幸福 |

## フィールド定義

### 必須フィールド

| フィールド | 型 | 説明 | 選択肢 |
|-----------|-----|------|--------|
| target_name | string | 対象の名前 | - |
| scale | enum | スケール | individual, company, family, country, other |
| period | string | 時期 | 例: "1985-1990", "2008年" |
| story_summary | string | 何が起きたか（100-200文字） | - |
| before_state | enum | 変化前の状態 | 下記参照 |
| trigger_type | enum | きっかけのタイプ | 下記参照 |
| action_type | enum | 取った行動 | 下記参照 |
| after_state | enum | 変化後の状態 | 下記参照 |
| before_hex | enum | 変化前の八卦 | 乾/坤/震/巽/坎/離/艮/兌 |
| trigger_hex | enum | きっかけの八卦 | 乾/坤/震/巽/坎/離/艮/兌 |
| action_hex | enum | 行動の八卦 | 乾/坤/震/巽/坎/離/艮/兌 |
| after_hex | enum | 変化後の八卦 | 乾/坤/震/巽/坎/離/艮/兌 |
| pattern_type | enum | パターン分類 | 下記参照 |
| outcome | enum | 結果 | Success, Failure, Mixed, PartialSuccess |
| source_type | enum | ソースタイプ | official, news, book, blog, sns, article |
| credibility_rank | enum | 信頼性ランク | S, A, B, C |

### オプションフィールド

| フィールド | 型 | 説明 |
|-----------|-----|------|
| logic_memo | string | 易経的な解釈メモ |
| free_tags | array | 自由タグ（キーワード） |

## 選択肢の詳細

### before_state / after_state
- 絶頂・慢心
- 停滞・閉塞
- 混乱・カオス
- 成長痛
- どん底・危機
- 安定・平和
- V字回復・大成功
- 縮小安定・生存
- 持続成長・大成功
- 安定成長・成功
- 変質・新生
- 現状維持・延命
- 迷走・混乱
- 崩壊・消滅

### trigger_type
- 外部ショック（市場変化、災害、競合の台頭など）
- 内部崩壊（不祥事、内紛、経営ミスなど）
- 意図的決断（戦略転換、リストラ、新規参入など）
- 偶発・出会い（偶然の出会い、チャンス到来など）

### action_type
- 攻める・挑戦（新規事業、M&A、海外進出など）
- 守る・維持（現状維持、防衛策など）
- 捨てる・撤退（事業売却、撤退、リストラなど）
- 耐える・潜伏（我慢、時期を待つなど）
- 対話・融合（交渉、提携、和解など）
- 刷新・破壊（組織改革、イノベーションなど）
- 逃げる・放置（問題回避、先送りなど）
- 分散・スピンオフ（分社化、多角化など）

### pattern_type
- Shock_Recovery（ショックからの回復）
- Hubris_Collapse（慢心からの崩壊）
- Pivot_Success（ピボットによる成功）
- Endurance（耐久・忍耐）
- Slow_Decline（緩やかな衰退）
- Steady_Growth（着実な成長）

## 出力形式

**必ずJSON配列として出力してください。**

```json
[
  {
    "target_name": "スティーブ・ジョブズ",
    "scale": "individual",
    "period": "1985-1997",
    "story_summary": "Appleを追放され、NeXTとPixarを設立。NeXTは商業的に失敗したが、Pixarは『トイ・ストーリー』で大成功。1997年にAppleに復帰し、復活の基盤を築いた。",
    "before_state": "絶頂・慢心",
    "trigger_type": "内部崩壊",
    "action_type": "攻める・挑戦",
    "after_state": "V字回復・大成功",
    "before_hex": "乾",
    "trigger_hex": "震",
    "action_hex": "離",
    "after_hex": "乾",
    "pattern_type": "Shock_Recovery",
    "outcome": "Success",
    "source_type": "book",
    "credibility_rank": "S",
    "logic_memo": "絶頂（乾）から衝撃（震）で追放され、才能を磨く期間（離）を経て、再び強さ（乾）へ。失敗と学びが後の成功の基盤となった。",
    "free_tags": ["Apple", "Pixar", "復活", "起業家"]
  }
]
```

---

## 注意事項

1. **推測で埋めない**: 情報が不足している場合は、そのフィールドを空にするか、抽出しない
2. **複数の変化がある場合**: 最も重要な3-5件に絞る
3. **八卦の選択**: 状況に最も近いものを選ぶ。迷ったら story_summary と整合性を取る
4. **credibility_rank**: Wikipedia=A, 公式資料=S, ニュース=A/B, ブログ=B/C
