# Phase 4: Q6表現の有用性検証（予測的妥当性）

**作成日**: 2026-03-09
**ステータス**: 設計段階（未着手）
**前提**: Phase 3 クローズ（2026-03-07）

---

## 仮説の再定式化

### 旧仮説（Phase 3）

> 実データの遷移グラフ G が Q6超立方体と同型である

**結果**: 測定器不適格のため判定不能。

Phase 3の検証過程で、アノテーションプロセス自体の根本的問題（純卦崩壊: 98.2%が8純卦に集中）が発見された。64卦フル空間を使用していないデータに対して同型性を検証すること自体が不適切であり、仮説の棄却/支持以前に測定器（ラベル生成プロセス）が不適格であった。

GPT-5.2（Codex）批評の要点:
- 「同型性」は強すぎる主張。比較可能な主張に落とすべき
- 測定器不適格が正しいクローズ理由

### 新仮説（Phase 4）

**H1（予測的有用性）**:
Q6表現（6bit hexagram encoding）は、メタデータのみのベースラインを超えて、変化事例の outcome_type / pattern_type を予測する能力を持つ。

**H2（相補性）**:
Q6表現は、テキスト特徴量（TF-IDF）に対して相補的な情報を提供する。

### 仮説の性質の違い

| 観点 | Phase 3（旧） | Phase 4（新） |
|------|--------------|--------------|
| 主張の強さ | 同型性（構造的同一性） | 予測的寄与（有用性） |
| 検証方法 | グラフ理論的比較 | 統計的モデル比較 |
| 失敗の意味 | Q6構造は無関係 | Q6表現は予測に寄与しない |
| 成功の意味 | データがQ6に従う | Q6表現は有用な特徴量 |
| 測定器への要求 | 64卦フル空間の正確なラベル | outcome/patternラベルの一貫性 |

## 検証フレームワーク

### 変数設計

**従属変数**（予測対象）:
- `outcome_type` — 変化事例の結果分類
- `pattern_type` — 変化パターンの分類

**独立変数**（特徴量群）:

| モデル | 特徴量 | 目的 |
|--------|--------|------|
| M0: Null | 多数決のみ | 下限 |
| M1: Metadata | scale, domain, era, source_type | ベースライン |
| M2: M1 + Q6 | M1 + hexagram 6bit + trigram pairs | H1検証 |
| M3: M1 + Text | M1 + TF-IDF（日本語要約） | テキストベースライン |
| M4: M1 + Text + Q6 | M3 + hexagram 6bit + trigram pairs | H2検証 |

### 比較手法

- **M1 vs M2**: nested model comparison（尤度比検定）→ H1の検証
- **M3 vs M4**: nested model comparison → H2の検証
- **評価指標**: 5-fold cross-validation, accuracy, macro-F1, AUC (one-vs-rest)
- **統計検定**: McNemar検定（モデル間の予測差異）、尤度比検定（nested model）

## 前提条件（Phase 4実行前に必要なもの）

Phase 3の教訓: 測定器が不適格な状態で検証を進めても判定不能になる。Phase 4ではこの轍を踏まない。

### 1. ゴールドセットによるラベル妥当性の確立

- **目標**: outcome_type, pattern_type の注釈者間一致度 Cohen's κ ≥ 0.6
- **手法**: 200件のゴールドセットを2名以上の独立アノテーターで付与
- **不合格時**: 注釈規約を改訂して再計測。κ < 0.4 なら当該ラベルは使用不可

### 2. 注釈規約の固定

- outcome_type, pattern_type の定義・境界事例・判断基準を文書化
- Phase 3で発見されたラベル多義性（action_typeの22ラベル中17が高多義性 JSD > 0.1）の教訓を反映
- 規約固定後のラベル変更は禁止（変更する場合はゴールドセット再計測）

### 3. 重複排除の完了

- 同一事例の重複エントリを排除
- 重複判定基準: title + entity + year の完全一致、またはテキスト類似度 > 0.9

### 4. ソース偏重の補正

- 現行データの source_type 分布: news 92%, book 4%, official 2%, blog 2%
- **対策**: 層化サンプリング、または逆確率重み付け（IPW）
- モデル評価時に source_type 層別の性能も報告

## 成功基準

### H1（予測的有用性）の判定

| 結果 | 条件 | 判定 |
|------|------|------|
| 支持 | M2 の accuracy が M1 を ΔAccuracy ≥ 2pp 上回り、かつ p < 0.01（尤度比検定） | Q6表現は予測に有用 |
| 弱い支持 | ΔAccuracy ≥ 1pp かつ p < 0.05 | 弱い有用性の示唆 |
| 不支持 | ΔAccuracy < 1pp または p ≥ 0.05 | Q6表現は予測に寄与しない |

### H2（相補性）の判定

| 結果 | 条件 | 判定 |
|------|------|------|
| 支持 | M4 の accuracy が M3 を ΔAccuracy ≥ 1pp 上回り、かつ Q6特徴量の部分相関 > 0 | Q6はテキストと独立した情報を持つ |
| 不支持 | ΔAccuracy < 1pp かつ 部分相関 ≈ 0 | Q6はテキスト情報の冗長な再表現 |

## 外部基準（External Validation）の設計

**追加日**: 2026-03-09
**背景**: Codex批評により、予測的有用性の検証が自己完結（同一データ内での交差検証のみ）にならないよう、外部基準を定義する必要が指摘された。

### 1. 時間的外部性（Temporal Out-of-Sample）

Q6表現が時間的に未知の事例に対しても予測力を持つかを検証する。

| 区分 | 条件 | 目的 |
|------|------|------|
| 訓練データ | period 開始年 ≤ 2022 の事例 | 過去データでモデルを学習 |
| テストデータ | period 開始年 ≥ 2023 の事例 | 時間的に未知のデータで評価 |

**評価方法**:
- 5-fold CV（Step B）に加え、時間分割による hold-out 評価を実施
- 訓練データで構築した M1, M2, M3, M4 をテストデータに適用
- M2 vs M1、M4 vs M3 の Δmacro-F1 を時間分割でも報告
- 時間分割結果がCV結果と乖離する場合、Q6表現の時間的安定性に疑義ありと報告

**制約**:
- 2023年以降の事例数が十分か（目安: 各クラス30件以上）を事前に確認
- 不足の場合、分割年を2021/2022に前倒しする

### 2. ドメイン外汎化（Cross-Domain Generalization）

Q6表現がドメインを超えて汎化するかを検証する。

| 区分 | scaleの値 | 目的 |
|------|----------|------|
| 訓練データ | company + individual | 事例数の多いドメインで学習 |
| テストデータ | country + family + other | 異なるドメインで評価 |

**評価方法**:
- 訓練ドメインで構築した M1, M2 をテストドメインに適用
- Q6表現の寄与度がドメインを超えて維持されるか
- ドメイン間で Δmacro-F1 の符号が反転する場合、Q6表現はドメイン固有の特徴を捉えているに過ぎないと判断

**制約**:
- テストドメイン（country, family, other）の事例数が少ない可能性
- 事例数が100件未満のドメインは個別評価から除外し、統合して評価

### 3. 外部データとの照合（将来の拡張）

Phase 4の範囲では実施しないが、将来の検証のために設計を記録する。

#### 3.1 公開データセットとの比較

- **企業倒産予測データセット**（例: UCI Machine Learning Repository の bankruptcy datasets）との照合
- 対象: scale = company かつ outcome_type に倒産/破綻を含む事例
- Q6表現を付与し、既存の財務指標ベースの予測モデルと比較

#### 3.2 独立測定可能な指標との照合

変化の成否を易経ラベルとは独立に測定可能な外部指標:

| 外部指標 | 対象 | 測定方法 |
|---------|------|---------|
| 株価変動 | company事例 | 事例期間前後の株価変動率（Yahoo Finance等） |
| 信用格付け変更 | company事例 | S&P/Moody's の格付け変更履歴 |
| GDP成長率 | country事例 | World Bank Open Data |
| 選挙結果 | country事例の政治系 | 公開選挙データ |

**検証の問い**: Q6表現が予測する outcome_type は、これらの独立指標と整合するか？

#### 3.3 実施条件

- Phase 4の確認的分析（H1, H2）が完了した後に実施
- 外部データの取得可能性を事前に調査
- 外部データとの照合結果は探索的分析として報告

## リスクと緩和策

| リスク | 影響 | 緩和策 |
|--------|------|--------|
| ゴールドセットでκ < 0.4 | Phase 4実行不可 | ラベル体系の再設計（カテゴリ統合・階層化） |
| サンプルサイズ不足（特定outcome_type） | 統計的検出力不足 | クラス統合、またはbinary化（成功/失敗） |
| Q6表現の情報量が本質的に低い | H1/H2共に不支持 | これ自体が有意義な結論。Q6は装飾的表現に留まる |
| 再アノテーション後も64卦の分布が偏る | モデル性能への影響 | 出現頻度による重み付け、または頻度閾値で除外 |
