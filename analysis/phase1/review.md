## 3. 発見された問題

### Issue #1: レポート内の事実誤認（重要度: HIGH）

**場所**: `report.md` 96-108行目

レポートの自己綜卦テーブルと注記に事実誤認がある:

```
| 28 | [0,1,1,1,1,0] | 沢風大過 |
| 61 | [0,1,1,1,1,0] | 風沢中孚 |   <-- 誤り
```

```
**注**: 卦28と卦61、卦27と卦62は同一ビット列の回文ペアである。  <-- 誤り
```

**実際の計算結果**:
- 卦28: [0,1,1,1,1,0]（沢風大過）-- 正しい
- 卦61: [1,1,0,0,1,1]（風沢中孚）-- レポートの[0,1,1,1,1,0]は誤り
- 卦27: [1,0,0,0,0,1]（山雷頤）-- 正しい
- 卦62: [0,0,1,1,0,0]（雷山小過）-- レポートの[1,0,0,0,0,1]は誤り

卦28と卦61、卦27と卦62は同一ビット列**ではない**。全64卦のビット列はユニークであることが検証で確認されている（`all_unique_bits: OK`）。注記の「同一ビット列の回文ペア」という記述は矛盾している。

**修正要求**: テーブルのビット列を正しい値に修正し、誤った注記を削除すること。

### Issue #2: 仕様との乖離 — DiGraph vs Graph（重要度: MEDIUM）

**場所**: `state_space_model.py` 167-198行目

仕様書 (`specs/phase1_spec.md` line 24) では `build_graph() -> nx.DiGraph` と定義されているが、実装は `nx.Graph`（無向グラフ）を返す。

レポートでは「1爻変化は双方向のため、無向グラフとして表現」と正当な理由が述べられており、数学的にはどちらも正しい。ただし、仕様との乖離は明示的に文書化されるべきである。

**推奨**: レポートの「既知の限界と仮定」セクションに「仕様書では`nx.DiGraph`を指定しているが、Q6は無向グラフであるため`nx.Graph`を使用した」旨を追記すること。

### Issue #3: 環境依存のフォント設定（重要度: LOW）

**場所**: `state_space_model.py` 33-49行目

macOSのヒラギノフォントパスがハードコードされている。Linux/Windows環境では日本語フォントが見つからずフォールバックする。可視化の再現性には影響しない（データは同一）が、見た目が異なる可能性がある。

**推奨**: フォント設定の環境依存性をレポートに明記するか、`matplotlib`の`rcParams`にフォールバックチェーンを設定すること。

### Issue #4: community-louvainの依存バージョン未記載（重要度: LOW）

**場所**: `state_space_model.py` 14行目

`community`（python-louvain）ライブラリのバージョンによってコミュニティ分割結果が変わる可能性がある。`random_state=42`でシード固定しているが、アルゴリズム実装の変更には対応できない。

**推奨**: 使用ライブラリとバージョンをレポートまたはrequirements.txtに記載すること。
