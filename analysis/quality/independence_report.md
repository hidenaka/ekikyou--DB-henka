## 3. 標本独立性への影響評価

### 3.1 重複による影響

- **完全一致ペアのリスク: 中** (26ペア)
  - 同一事象の多重記録の可能性が高い。検定の標本独立性前提を直接破壊する
- **高リスクニアミス: 57ペア**
  - 表記揺れによる同一entityかつ類似内容。手動レビュー必須
- **中リスクニアミス: 1774ペア**
  - 同一entityの別フェーズ/別イベントの可能性。独立性判定基準で仕分け
- **低リスクニアミス: 5088ペア**
  - entity名の部分一致のみ。内容は異なる。多くは独立事例と推定

### 3.2 集中度による影響

- **リスク: 低**
- 上位10 entityが全体の 0.49% を占める
- 集中度は低い。ただし173 entityが複数事例を持つため、entity-levelのクラスタリングは推奨

### 3.3 同一entityの複数事例が独立と見なせる条件（提案）

以下の **全て** を満たす場合、同一entityの2事例は独立標本として扱える:

1. **時間的分離**: 2事例のperiodが5年以上離れている
2. **フェーズの相違**: event_phase / pattern_type が異なる
3. **トリガーの相違**: trigger_type が異なる（同一トリガーの連鎖でない）
4. **要約の非類似**: story_summary の Jaccard類似度が 0.3 未満

上記を満たさない場合:
- entity_idによるクラスタリング + cluster-robust standard errorsの適用
- または、entityごとに最も情報量の多い1事例を代表として選択

## 5. GPT-5.2批評への回答

### 指摘: 「同一事象の多重ソースは独立標本ではない」

**現状の影響度: 限定的（ただし対処必要）**

- 完全一致（明確な重複）: 26ペア → 全体の 0.46%
- 高リスクニアミス: 57ペア → 手動確認で重複判定が必要
- entity集中度: Top10が0.49% → 低い
- ユニークentity率: 10911/11084 = 98.4%

**結論**: データの大部分（96%+のentity）は1事例のみで、標本独立性の問題は限定的。
ただし完全一致26ペアは確実に対処が必要。ニアミスの高リスク分は手動確認を推奨。
検定時にはentity-levelのクラスタリングを適用すれば、批評の懸念に十分対応できる。
