## 3. 標本独立性への影響評価

### 3.1 重複による影響

- **完全一致ペアのリスク: 中**
  - 同一事象の多重記録の可能性が高い。検定の標本独立性前提を直接破壊する
- **類似重複のリスク: 低**
  - 同一事象を別角度で記述した可能性。手動レビューが必要
- **ニアミスのリスク: 要確認**
  - 表記揺れによる同一entity（例: 'トヨタ' vs 'トヨタ自動車'）の可能性

### 3.2 集中度による影響

- **リスク: 低**
- 上位10 entityが全体の 0.49% を占める

### 3.3 同一entityの複数事例が独立と見なせる条件（提案）

以下の **全て** を満たす場合、同一entityの2事例は独立標本として扱える:

1. **時間的分離**: 2事例のperiodが5年以上離れている
2. **フェーズの相違**: event_phase / pattern_type が異なる
3. **トリガーの相違**: trigger_type が異なる（同一トリガーの連鎖でない）
4. **要約の非類似**: story_summary の Jaccard類似度が 0.3 未満

上記を満たさない場合:
- entity_idによるクラスタリング + cluster-robust standard errorsの適用
- または、entityごとに最も情報量の多い1事例を代表として選択
