## 1. 研究課題

Q6表現（易経64卦の6bit encoding）は、変化事例の outcome_type / pattern_type の予測において、メタデータのみ・テキスト特徴量のみのベースラインを超える寄与を持つか？

---

## 2. 仮説

### 確認的仮説（Confirmatory）

**H1（予測的有用性）**:
Q6表現の追加は、メタデータベースライン（M1）に対して ΔAccuracy ≥ 2pp の改善をもたらす（α = 0.01、Holm-Bonferroni補正後）。

**H2（相補性・独立情報）**:
Q6表現は、テキスト特徴量（TF-IDF）に対して独立な情報を持つ。M4（M3 + Q6）がM3に対して有意な改善を示し、Q6特徴量の部分相関が有意である（p < 0.01、Holm-Bonferroni補正後）。

**H0（帰無仮説）**:
Q6表現の追加は予測性能を改善しない。M2とM1の差、およびM4とM3の差は統計的に有意でない。

### 仮説の判定基準

| 仮説 | 採択条件 | 棄却条件 |
|------|---------|---------|
| H1 | M2 vs M1: Δmacro-F1 ≥ 2pp かつ p < 0.01（補正後） | Δmacro-F1 < 2pp または p ≥ 0.01 |
| H2 | M4 vs M3: Δmacro-F1 > 0 かつ部分相関検定 p < 0.01（補正後） | 部分相関 ≈ 0 または p ≥ 0.01 |

---

## 3. 評価指標

### 主要評価指標（Primary）

**macro-F1 のΔ（差分）** — モデル間の macro-F1 スコアの差。
主要指標として macro-F1 を採用する理由: クラス不均衡下で少数クラスの予測力も反映するため。

### 副次評価指標（Secondary）

| 指標 | 用途 |
|------|------|
| Accuracy | 全体的な正解率の確認 |
| AUC (one-vs-rest) | 閾値非依存の識別力評価 |
| Cohen's d | 効果量の報告（CV fold間のスコア差の標準化効果量） |

### 指標の報告ルール

- 主要指標（macro-F1 Δ）で仮説判定を行う
- 副次指標は補助的に報告するが、仮説判定には使用しない
- 主要指標と副次指標の結果が矛盾する場合は、その矛盾を明示的に報告する

## 4. サンプルサイズ正当化

### 現在のデータサイズ

- 総事例数: 11,336件（重複フラグ付き事例を除外後の有効数は変動しうる）

### 必要サンプルサイズの算出

効果量 d = 0.3（small-medium）を検出するために必要なサンプルサイズ:

```
n = (Z_{α/2} + Z_β)² × 2 / d²

α = 0.01 (両側), Z_{α/2} = 2.576
β = 0.20 (検出力 80%), Z_β = 0.842

n = (2.576 + 0.842)² × 2 / 0.3²
n = (3.418)² × 2 / 0.09
n = 11.683 × 2 / 0.09
n = 259.6 ≈ 260 （各群）
```

### 現データでの検出力

現在のデータサイズ（n ≈ 11,336）での検出力:

```
d_detectable = Z_{α/2} + Z_β) × √(2/n)
             = 3.418 × √(2/11336)
             = 3.418 × 0.01328
             = 0.0454

検出力計算（d=0.3, n=11,336の場合）:
Z = d × √(n/2) - Z_{α/2}
  = 0.3 × √(5668) - 2.576
  = 0.3 × 75.29 - 2.576
  = 22.587 - 2.576
  = 20.011

→ 検出力 ≈ 1.0（d=0.3の効果量は十分に検出可能）
```

### 結論

現在のデータサイズ（11,336件）は、small-medium効果量（d = 0.3）の検出に十分な検出力（> 0.99）を持つ。ただし、クラス別のサンプルサイズが極端に小さいカテゴリでは検出力が低下する可能性がある。クラス別サンプルサイズが30未満のカテゴリは分析から除外するか、上位カテゴリに統合する。

## 5. 分析パイプライン（固定）

以下のパイプラインは事前登録時点で固定する。順序・手法の変更は禁止。

### Step A: データクリーニング

- 重複フラグ付き事例を除外（重複判定基準: title + entity + year の完全一致、またはテキスト類似度 > 0.9）
- クラス別サンプルサイズが30未満のカテゴリは除外または統合（統合ルールは注釈規約に従う）

### Step B: 交差検証設計

- **層化**: source_type による層化（news, book, official, blog, other）
- **分割**: 5-fold cross-validation × 10 repeats（計50回の評価）
- **乱数シード**: 42（固定）
- **層化変数**: source_type × 従属変数のクロス

### Step C: ネストモデル比較

以下の順序でモデルを構築する:

| モデル | 特徴量 | 比較対象 |
|--------|--------|---------|
| M0: Null | 多数決のみ | — |
| M1: Metadata | scale, domain, era, source_type | M0 |
| M2: M1 + Q6 | M1 + hexagram 6bit + trigram pairs | M1 |
| M3: M1 + Text | M1 + TF-IDF（日本語要約） | M1 |
| M4: M1 + Text + Q6 | M3 + hexagram 6bit + trigram pairs | M3 |

- 分類器: ロジスティック回帰（L2正則化、C はネストCV内で選択）
- TF-IDFパラメータ: max_features=5000, min_df=5, sublinear_tf=True

### Step D: 統計検定

- **H1検定**: M1 vs M2 の尤度比検定（likelihood ratio test）
- **H2検定**: M3 vs M4 の尤度比検定 + Q6特徴量の部分相関検定
- **補助検定**: McNemar検定（モデル間の予測パターンの差異）
- **多重比較補正**: Holm-Bonferroni法（H1, H2 の2仮説に対して適用）

### Step E: クラスター補正

- **entity-level cluster-robust standard errors** を適用
- 同一entity（企業・人物等）に属する複数事例は独立でないため、entityレベルでクラスタリングし、標準誤差をクラスター補正する
- クラスター数が50未満の場合は、wild cluster bootstrap（Webb 6-point）を適用

## 6. 停止条件（Go/No-Go Gates）

分析は段階的に進行し、各Gateを通過しない限り次段階に進まない。

### Gate 1: パイロットアノテーション品質

- **対象**: ゴールドセット200件に対するアノテーター間一致度
- **測定**: 全4 trigramフィールド（upper_trigram, lower_trigram, changing_to_upper, changing_to_lower）の平均Cohen's κ
- **GO条件**: 平均κ ≥ 0.5
- **不合格時**: 注釈規約を改訂して再試行（最大3回）
- **完全不合格**: 3回改訂後もκ < 0.4 → 当該フィールドは分析から除外

### Gate 1b: 条件付き通過（Gate 1 不合格時の代替）

- **条件**: 平均κ ≥ 0.4 かつ Gwet's AC1 ≥ 0.5
- **意味**: κが有病率の偏りで過小評価されている可能性がある場合の救済
- **通過時**: Gate 1b通過を明示的に報告し、κの限界を議論する

### Gate 2: 評価セットアノテーション品質

- **対象**: 全件アノテーション前の評価セット（500件）
- **GO条件**: outcome_type, pattern_type の Cohen's κ ≥ 0.6
- **不合格時**: 注釈規約を再改訂。κ < 0.5 → Phase 4は実行不可と結論

### Gate 3: 主要仮説判定

- **GO条件（H1採択）**: M2のM1に対する Δmacro-F1 ≥ 2pp かつ p < 0.01（Holm-Bonferroni補正後）
- **不合格時**: H1棄却。Q6表現は予測に寄与しないと結論
- **弱い支持**: Δmacro-F1 ≥ 1pp かつ p < 0.05 の場合、「弱い有用性の示唆」として報告するが、H1は採択しない

### 全Gate不通過時の結論

全Gateを通過できない場合、以下のいずれかを結論とする:
- Gate 1/1b 不通過: 「Q6表現のラベルが測定に耐えない品質であり、検証不能」
- Gate 2 不通過: 「outcome/pattern ラベルの品質が検証に不十分であり、Phase 4は実行不可」
- Gate 3 不通過: 「Q6表現は outcome/pattern の予測に統計的に有意な寄与を持たない」

## 7. 多重比較補正

### 適用範囲

確認的仮説は2つ（H1, H2）であるため、Holm-Bonferroni法を適用する。

### 手順

1. H1, H2 それぞれの p値を算出
2. 小さい方の p値に対して α/2 = 0.005 で判定
3. 小さい方が有意の場合、大きい方を α/1 = 0.01 で判定
4. 小さい方が非有意の場合、両方とも非有意と判定

### 実効α水準

- 個別仮説: α = 0.01
- 族全体（familywise）: α = 0.01（Holm-Bonferroni補正により制御）
