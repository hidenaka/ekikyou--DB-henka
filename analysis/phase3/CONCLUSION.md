# Phase 3 同型性検証 -- 最終結論

**クローズ日**: 2026-03-07
**Phase期間**: 2026-02 ~ 2026-03
**使用事例数**: 9,228件（除外: 2,108件）
**バージョン履歴**: v2.0（初期検証） -> v2.1（純卦崩壊発見） -> v2.2（確率的シミュレーション） -> v2.3（100件LLM再アノテーション）

---

## エグゼクティブサマリー

1. **Q6隣接遷移仮説は判定不能**。測定器（LLMによるhexagramラベル生成プロセス）が不適格であったため、仮説の棄却/支持以前に検証が成立しない。純卦崩壊（98.2%が8純卦に集中）により64卦フル空間が使用されておらず、Q6超立方体との同型性を評価する前提条件を欠いていた。
2. **v2.0で報告した「偶数パリティ対称性」「上下同型変換」は、純卦崩壊の機械的アーティファクトであることが4つの独立検証で確定した。**
3. **マクロ対称性（スペクトル・錯卦・コミュニティ）には弱い部分反映が見られるが、Q6固有ではなく(Z2)^6空間の一般的対称性で説明可能な範囲にとどまる。**
4. **Phase 4へ移行**: 仮説を「同型性（構造的同一性）」から「Q6表現の予測的有用性」に再定式化し、比較可能な主張として検証する。詳細は `analysis/phase4_design.md` を参照。

## 純卦崩壊の発見と影響

### 発見内容

LLMによるhexagram付与プロセスが64卦のフル空間を使わず、8個の純卦（上卦==下卦: 乾為天/坤為地/坎為水/離為火/震為雷/巽為風/艮為山/兌為沢）にプロトタイプ的に圧縮していた。classical_before/after_hexagramの98.2%が純卦に集中。

| カテゴリ | 件数 | 割合 | 対角率 |
|----------|------|------|--------|
| both_pure（両方純卦） | 9,007 | 97.6% | 100.00% |
| one_pure（片方のみ） | 110 | 1.2% | 0.00% |
| both_nonpure（両方非純卦） | 111 | 1.2% | 9.01% |

source_type別: news/blog = 99-100%純卦、book = 88-90%、official = 55-57%。

### 影響

純卦は上卦==下卦なので、2つの純卦間の遷移は定義上 Delta_lower = Delta_upper を満たす。したがって:

1. **偶数パリティ偏向（98.6%）**: 純卦崩壊の機械的帰結であり、データの構造的特徴ではない
2. **上下同型変換（97.8%）**: 同上
3. **ハミング距離分布の偶数偏重**: 純卦間遷移では必然的に偶数ハミング距離となる

v2.0で「構造的発見」と報告した偶数パリティ・上下同型変換は、退化解を誤認した擬似パターンであった。

## 最終結論

### 確定事項

1. **Q6隣接遷移仮説: 測定器不適格のため判定不能**。測定器（hexagramラベル生成）が64卦フル空間を使用していなかった（98.2%が8純卦に退化）ため、同型性の検証自体が成立しない。Test A（overlap 5.2%）/ Test B（距離1遷移 0.13%）の結果はこの退化の帰結であり、仮説の棄却根拠としては不十分。
2. **偶数パリティ/上下同型変換: 純卦崩壊アーティファクト**。4つの独立検証（パイロットLLM n=15、確率シミュレーション n=11,336x1,000、純卦分析、100件LLM再アノテーション）で確定。
3. **マクロ対称性: 弱い部分反映**。スペクトル類似度（p<0.001）と錯卦対称性（p=0.001）でH0を棄却したが、(Z2)^6空間の一般的性質で説明可能な範囲。Q6固有の対応とは断定できない。
4. **64卦空間の未活用**: 現行アノテーションは8純卦に退化しており、56個の非純卦（上卦!=下卦）の組合せ意味論がほぼ活用されていない。

### 未解決事項

- **対角率42%の残余構造**: 100件LLM再アノテーションで対角率42%がランダム基準(30.9%)を上回った。LLMによる意味的アノテーションが上下卦間に構造を生むのか、サンプルサイズ不足による揺らぎか、全件再アノテーション後に検証が必要。

### Phase 3の総括的意義

Phase 3はQ6同型性仮説の検証から出発し、その過程でアノテーションプロセス自体の根本的問題（純卦崩壊）を発見した。これはPhase 3の「失敗」ではなく、データ品質の致命的な問題を可視化した成果である。

正確なクローズ理由は「棄却」ではなく「測定器不適格のため判定不能」である。GPT-5.2（Codex）批評が指摘した通り、「同型性」は強すぎる主張であり、測定器が不適格な状態では支持も棄却もできない。

### Phase 4への移行

Phase 3の結論を受け、仮説を再定式化した:

- **旧（Phase 3）**: 実データの遷移グラフ G が Q6超立方体と同型である → 判定不能
- **新（Phase 4）**: Q6表現（6bit hexagram encoding）が変化事例の outcome/pattern を予測する能力を持つか → 比較可能な主張として検証

Phase 4の設計文書: `analysis/phase4_design.md`

## 今後への提言

### 1. 全件再アノテーション（最優先）

現行の8純卦への退化を解消するため、全9,534件の2段階八卦再アノテーション（内卦・外卦を独立に選択）が必要。100件再アノテーションで確立した手法（Claude 4バッチ並列、reasoning付き）をスケールアップする。

### 2. 再アノテーション後のPhase 3テスト再実行

64卦フル空間を活用したデータで test_a ~ test_e を再実行し、Q6同型性仮説を改めて検証する。対角率42%の残余構造が統計的に有意か否かも、この段階で確定できる。

### 3. BacktraceEngine / フィードバックエンジンへの知見活用

- **純卦崩壊の教訓**: hexagramフィールドに基づく遷移分析は、再アノテーション完了まで信頼性が低い。BacktraceEngineは八卦（before_hex/after_hex）レベルでの分析を優先すべき。
- **マクロ対称性の限定的活用**: 錯卦対称性（ビット反転）は弱いながらもデータに反映されている。フィードバックエンジンで「対称的な変化パターン」の検索に補助的に使用可能。
- **ラベル多義性**: action_typeの22ラベル中17が高多義性（JSD > 0.1）。フィードバックエンジンでのaction_type使用時にはスケール文脈の注入が必須。

### 4. アノテーションガイドラインの整備

100件再アノテーションで純卦率0%・ユニーク卦37/64を達成した手法（独立八卦選択 + reasoning付き）を標準プロトコルとして文書化し、今後のデータ拡充に適用する。
