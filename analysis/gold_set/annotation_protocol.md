# ゴールドセット アノテーション規約 v2.0

**改訂履歴**: v1.0 → v2.0 (2026-03-09) Codex批評に基づき決定木・境界事例・判定不能カテゴリ・反例を追加

## 目的

cases.jsonlの事例テキスト（before_state, after_state, story_summary等）から、
**意味ベース**で内卦（lower trigram）・外卦（upper trigram）を選定する。

既存の確率テーブルベースの「合成ラベル」を置き換える、信頼性の高い正解データを構築する。

---

## 2段階アノテーション方式

1段階目と2段階目で独立にアノテーションし、一致率（Cohen's κ + Gwet's AC1）で品質を測定する。

### 1段階目: 初回アノテーション
- before_state / after_state / story_summary を読み、下記の決定木に従って八卦を選定
- reasoning（選定理由）を必ず記述

### 2段階目: 検証アノテーション
- 1段階目の結果を見ずに、同じ事例を別途アノテーション
- 不一致箇所は裁定（adjudication）で最終決定

---

## 内卦・外卦の選択基準

### 内卦（lower trigram）: 主体の内的状態・本質

**判定対象**: 事例の主体（企業・個人・国家等）が内面的にどのような状態にあるか。
外部から見えない内部の実態、組織の本質的な状態、意思決定者の心理状態。

- 表面的な行動ではなく、**内部で何が起きているか**を見る
- 「なぜそうなったか」の根底にある状態

### 外卦（upper trigram）: 外的環境・状況・対外的表現

**判定対象**: 主体を取り巻く環境、外部からの圧力、市場状況、社会的文脈、
主体が外部に見せている姿。

- 主体の内部状態とは独立した外部の力学
- 「周囲から見てどう映っているか」「環境はどう動いているか」

---

## 決定木（判定優先規則）

### Step 1: テキストから主体の行動/状態を特定

以下のフィールドを順に確認する:
1. `story_summary` — 事例の全体像を把握
2. `before_state` — 変化前の状態ラベル
3. `after_state` — 変化後の状態ラベル
4. `action_type` — とられた行動
5. `trigger_type` — 変化のきっかけ

### Step 2: 内卦選択 — 主体の内的動機・本質を以下の優先順で判定

**判定は上から順に試行し、最初に該当した卦を選択する**。

| 優先順 | 条件（テキストから読み取れる内的状態） | 内卦 |
|--------|----------------------------------------|------|
| 1 | 主体が強い意志で積極的に拡大・主導している | ☰ 乾 |
| 2 | 主体が受容的・従順で基盤形成に徹している | ☷ 坤 |
| 3 | 主体が突発的な決断・行動を起こした / 内部に衝撃的変化 | ☳ 震 |
| 4 | 主体が柔軟に適応し漸進的に変化している | ☴ 巽 |
| 5 | 主体が深刻な困難・リスクに直面している | ☵ 坎 |
| 6 | 主体に明確なビジョン・情熱・透明性がある | ☲ 離 |
| 7 | 主体が意図的に停止・蓄積・内省している | ☶ 艮 |
| 8 | 主体が成果を享受・交流・開放している | ☱ 兌 |

### Step 3: 外卦選択 — 外部環境を同じ優先順で判定

| 優先順 | 条件（テキストから読み取れる外的環境） | 外卦 |
|--------|----------------------------------------|------|
| 1 | 環境が成長・拡大を後押し / 上昇トレンド | ☰ 乾 |
| 2 | 環境が安定的で大きな変化がない / 成熟市場 | ☷ 坤 |
| 3 | 外部から突発的衝撃 / 市場の急変動 | ☳ 震 |
| 4 | 環境が緩やかに変化中 / トレンドの漸進的浸透 | ☴ 巽 |
| 5 | 環境が厳しく危険 / 不況・規制強化 | ☵ 坎 |
| 6 | メディア注目 / 情報公開 / 社会の関心集中 | ☲ 離 |
| 7 | 環境が膠着 / 市場停滞 / 障壁 | ☶ 艮 |
| 8 | 市場が歓迎的 / 協力・提携の機会 | ☱ 兌 |

### Step 4: 競合時のルール（2つ以上該当する場合）

1. **テキスト中の最も顕著な特徴を優先**: story_summaryで最も多くの記述量が割かれている側面を優先する
2. **具体的記述 > 抽象的ラベル**: 「売上が3倍に成長」（乾）は「安定している」（坤）より具体的であり優先
3. **行動の結果ではなく状態を見る**: action_typeが「攻める」でも、内部が疲弊していれば坎を選ぶ
4. **before_stateラベルとの整合性**: state_labelは参考情報であり最終決定権を持たないが、テキスト証拠が拮抗する場合はstate_labelに従う
5. **内卦と外卦の独立性**: 内と外は別の判断。両方同じ卦になることもあるが、「迷ったから同じにした」は不可

---

## 八卦の操作的定義

### ☰ 乾（けん / Qian）: 強い推進力・積極的拡大・リーダーシップ発揮

**内卦として選ぶ場合**:
- 主体が強い意志と自信を持ち、主導権を握っている
- 積極的に拡大・成長を志向している
- リーダーシップが明確に発揮されている
- エネルギーが充溢し、前進する力がある

**外卦として選ぶ場合**:
- 環境が成長・拡大を後押ししている
- 市場や社会が上昇トレンドにある
- 外部から強い指導力・権威が働いている
- 競争が活発で積極的な行動が求められる環境

**典型例**: 急成長中の企業、カリスマ経営者の改革、好況市場への参入

---

### ☷ 坤（こん / Kun）: 受容・従順・基盤形成・静かな蓄積

**内卦として選ぶ場合**:
- 主体が受動的・受容的な姿勢をとっている
- 基盤づくり・地道な蓄積フェーズにある
- 指示や環境に従順に従っている
- 目立った主張をせず、支える側に回っている

**外卦として選ぶ場合**:
- 環境が安定的で大きな変化がない
- 市場が成熟し、急激な変動がない
- 支援的・養育的な社会環境
- 受け入れる体制・土壌が整っている

**典型例**: 創業準備期、地道な組織基盤構築、安定市場での堅実経営

---

### ☳ 震（しん / Zhen）: 衝撃・突発的変化・新規行動の開始

**内卦として選ぶ場合**:
- 主体が突発的な決断・行動を起こした
- 内部に衝撃的な変化が生じた（不祥事発覚、方針転換等）
- 新しい事業・取り組みを開始した
- 現状を打破する動きが内部から発生

**外卦として選ぶ場合**:
- 外部から突発的な衝撃が来た（天災、規制変更、競合の急変等）
- 市場に大きな変動が発生
- 予期せぬイベントが環境を揺るがした
- テクノロジーの破壊的変化

**典型例**: 突然の経営危機、業界の地殻変動、スタートアップの立ち上げ

---

### ☴ 巽（そん / Xun）: 浸透・柔軟な適応・漸進的影響

**内卦として選ぶ場合**:
- 主体が柔軟に適応しながら少しずつ変化している
- じわじわと浸透するような戦略をとっている
- 風のように柔軟で、抵抗を避けて入り込む
- 漸進的な改善・改革を進めている

**外卦として選ぶ場合**:
- 環境の変化がゆっくりだが確実に進行している
- 社会的なトレンドが徐々に浸透中
- 規制や文化の漸進的な変化
- 情報や影響が緩やかに広がっている

**典型例**: DX推進、企業文化の漸進的変革、市場浸透戦略

---

### ☵ 坎（かん / Kan）: 困難・リスク・試練・流動的不安定

**内卦として選ぶ場合**:
- 主体が深刻な困難に直面している
- 内部に構造的な問題・リスクを抱えている
- 財務危機、組織崩壊、信頼喪失の状態
- 出口が見えない試練の渦中にある

**外卦として選ぶ場合**:
- 環境が厳しく危険に満ちている
- 市場の暴落、不況、パンデミック等の外的危機
- 規制強化や訴訟リスクが高い
- 競合環境が苛烈で生存が脅かされている

**典型例**: 経営危機、不祥事後の信頼回復期、不況下のサバイバル

---

### ☲ 離（り / Li）: 明晰・可視化・注目・情熱

**内卦として選ぶ場合**:
- 主体の状態が外部から明確に見えている（透明性）
- 知性・分析力・ビジョンの明確さが中核にある
- 情熱や使命感が内部を動かしている
- アイデアや技術が輝きを放っている

**外卦として選ぶ場合**:
- メディアの注目を集めている
- 情報が公開・可視化されている
- 社会の関心が集中している
- 評価・審査の目にさらされている

**典型例**: IPO、メディア露出の増加、技術革新の可視化、炎上・注目

---

### ☶ 艮（ごん / Gen）: 停止・蓄積・内省・現状維持

**内卦として選ぶ場合**:
- 主体が意図的に立ち止まっている
- 内省・振り返りのフェーズにある
- 蓄積・準備期間として静かにしている
- これ以上進まないという判断をした

**外卦として選ぶ場合**:
- 環境が膠着し、動きが止まっている
- 市場が停滞・成熟し、成長余地が限られている
- 規制や障壁が前進を阻んでいる
- 社会的に「待ち」の状態

**典型例**: 事業撤退の判断、組織の再編前の静止期、市場成熟期

---

### ☱ 兌（だ / Dui）: 喜び・交流・開放・成果の享受

**内卦として選ぶ場合**:
- 主体が成果を実感し、満足・喜びの状態にある
- コミュニケーション・対話を重視している
- オープンで開放的な組織文化
- 人とのつながり・交流が中核的価値

**外卦として選ぶ場合**:
- 市場や社会が歓迎的・好意的
- 顧客や株主から高い評価を受けている
- 協力・提携の機会に恵まれている
- 祝祭的・楽観的なムード

**典型例**: 好業績発表、戦略的提携、ブランド確立後の享受期

---

## 判定手順（決定木に従った実行手順）

### Step 1: テキストを読む

1. `story_summary` — 事例の全体像を把握
2. `before_state` — 変化前の状態ラベル
3. `after_state` — 変化後の状態ラベル
4. `action_type` — とられた行動
5. `trigger_type` — 変化のきっかけ

### Step 2: Before側の内卦（lower）を選ぶ

決定木の優先順位表を上から順に試行する:
1. 積極的拡大行動が見られるか → 乾
2. 受容・基盤形成に徹しているか → 坤
3. 突発的行動開始があったか → 震
4. 漸進的浸透・適応をしているか → 巽
5. 困難・リスクに直面しているか → 坎
6. 明確なビジョン・情熱があるか → 離
7. 停止・蓄積・内省しているか → 艮
8. 交流・成果を享受しているか → 兌

**2つ以上該当する場合**: Step 4の競合ルールに従う。

### Step 3: Before側の外卦（upper）を選ぶ

同じ決定木の優先順位表を外部環境に適用する。内卦の選択結果に影響されず独立に判定する。

### Step 4: reasoning を記述

- なぜその八卦を選んだか
- テキストのどの部分が根拠か
- 迷った場合は候補と棄却理由を明記

### Step 5: After側の内卦・外卦を同様に選定

### Step 6: 整合性チェック

- Before → After の変化が story_summary の物語と整合するか確認
- 内卦と外卦が**異なる八卦**であることが多いはず（純卦は例外的）
  - 純卦率の目標: Before ≤ 40%, After ≤ 15%
- 既存の before_hex / after_hex（合成ラベル）との比較は参考程度にとどめる

### Step 7: uncertainフラグの判定

以下のいずれかに該当する場合、`uncertain: true` を付与する（詳細は「判定不能カテゴリ」参照）。

## 境界事例の処理規則

### 境界事例1: 「困難の中の積極性」— 坎 vs 乾

**事例パターン**: 経営危機下で大胆な改革を断行
- **判定**: 内卦は坎（困難に直面している現状が本質）、外卦は状況次第
- **理由**: 積極的行動（乾）は action_type に反映される属性であり、内的「状態」としては困難の中にいる
- **例外**: テキストが「危機を楽しんでいる」「ピンチをチャンスと捉え意気揚々」と明示的に記述している場合のみ乾

### 境界事例2: 「安定 vs 停滞」— 坤 vs 艮

**事例パターン**: 業績は安定しているが成長が止まっている
- **判定**: 主体が現状を**意図的に維持**しているなら艮、**環境に従っている**なら坤
- **識別基準**: 「判断」の有無。「維持する決断をした」→ 艮。「特に何もしていない」→ 坤
- **テキストに判断の記述がない場合**: 坤をデフォルトとする

### 境界事例3: 「漸進的変化 vs 受容」— 巽 vs 坤

**事例パターン**: ゆっくり変化しているが、主体的か受動的か不明
- **判定**: 変化の**方向性が明確**なら巽（意図的な浸透）、方向性が**不明確**なら坤（受容）
- **識別基準**: 「戦略」「計画」「段階的に」等の意図的語彙があれば巽

### 境界事例4: 「注目 vs 衝撃」— 離 vs 震（外卦）

**事例パターン**: 突然メディアに大きく取り上げられた
- **判定**: **継続的な**注目なら離、**一時的な衝撃性**が主なら震
- **識別基準**: 注目の持続期間。「連日報道」→ 離。「突如として」→ 震
- **両方の要素がある場合**: テキスト中の記述量が多い方を選択

### 境界事例5: 「内省 vs 困難」— 艮 vs 坎

**事例パターン**: 事業縮小して立ち止まっている
- **判定**: 縮小が**自発的な戦略**なら艮、**外部圧力による不可避な縮小**なら坎
- **識別基準**: 「選択と集中」→ 艮。「資金繰り悪化で」→ 坎

### 境界事例6: 「交流 vs 積極拡大」— 兌 vs 乾

**事例パターン**: 提携・M&Aを通じて急拡大
- **判定**: 拡大の**手段が交流・協力**でも、内的ドライバーが拡大志向なら乾
- **識別基準**: 「支配的な拡大」→ 乾。「Win-Winの協力関係」→ 兌

## 判定不能（uncertain）カテゴリ

### 定義

テキスト情報が不十分で、決定木を適用しても内卦または外卦を合理的に決定できない場合に `uncertain: true` を付与する。

### uncertainフラグを付ける条件

1. **story_summaryが2文以下**で、具体的な内的状態/外的環境の手がかりがない
2. **決定木の候補が3つ以上同等に該当**し、テキストからの識別情報が皆無
3. **内卦と外卦の区別が不可能**: テキストが内面と外面を混同して記述しており分離不能
4. **テキストが抽象的すぎる**: 「変化があった」のみで具体的な方向性が不明

### uncertainフラグの付与ルール

- `uncertain: true` を付与しても、**最も可能性の高い卦は選択する**（空欄にしない）
- reasoning に不確実性の理由を明記する
- **乱用防止**: 全体の10%以下に制限。超過した場合は規約の見直しが必要

### フォーマット

```json
{
  "before_lower": "坎",
  "before_upper": "震",
  "before_uncertain": true,
  "before_reasoning": "story_summaryが1文のみで内的状態の手がかりが不十分。before_state=どん底・危機から坎を推定したが確信度は低い"
}
```

## 反例（ダメな付与例）

### 反例1: state_labelへの機械的マッピング

```
before_state: "どん底・危機"
→ 内卦: 坎、外卦: 坎（坎為水）
```

**なぜダメか**: state_labelだけを見て八卦を決定している。story_summaryを読めば「危機だが社長は強い意志で改革を主導」とあるかもしれない。その場合、内卦は乾が適切。**テキスト全体から判断すること。**

### 反例2: 内卦と外卦を区別せず同じ卦にする

```
story_summary: "業績不振の中、政府の規制緩和が追い風に"
→ 内卦: 坎、外卦: 坎
```

**なぜダメか**: 内部は困難（坎）だが、外部環境は規制緩和で好転している。外卦は巽（漸進的変化）や乾（拡大を後押し）が適切。内外を独立に判断していない。

### 反例3: action_typeを内卦の根拠にする

```
action_type: "攻める・挑戦"
→ 内卦: 乾
```

**なぜダメか**: action_typeは主体が「とった行動」であり、内的「状態」ではない。攻めに出た動機が「追い詰められたから」であれば、内的状態は坎。行動ではなく状態を見ること。

### 反例4: uncertainフラグの乱用

```
story_summary: "トヨタがハイブリッド車戦略で市場を席巻した"
→ uncertain: true, 理由: "乾と巽で迷った"
```

**なぜダメか**: テキストは十分に具体的。「市場を席巻」は積極的拡大（乾）を明確に示している。2択で迷うだけではuncertainにならない。決定木の優先順位と競合ルールで解決すること。

## 品質指標

| 指標 | 目標値 | 説明 |
|------|--------|------|
| Cohen's κ（Annotator A vs B） | ≥ 0.60 | 実質的一致（substantial agreement） |
| Gwet's AC1（Annotator A vs B） | ≥ 0.65 | 偏り耐性のある一致指標 |
| 純卦率（Before） | ≤ 40% | 上下卦が同一の事例の割合 |
| 純卦率（After） | ≤ 15% | 同上 |
| ユニーク卦数（Before） | ≥ 25/64 | 64卦のうち何種類が出現するか |
| ユニーク卦数（After） | ≥ 25/64 | 同上 |
| reasoning記入率 | 100% | 全件に選定理由を記述 |
| uncertain率 | ≤ 10% | 判定不能フラグの割合 |

### 複合判定基準（κ単独判定をしない）

κ ≥ 0.60 かつ AC1 ≥ 0.65 → **PASS**
κ ≥ 0.60 かつ AC1 < 0.65 → **CONDITIONAL PASS**（偏りの可能性を検討）
κ < 0.60 かつ AC1 ≥ 0.65 → **CONDITIONAL PASS**（低頻度カテゴリの影響を検討）
κ < 0.60 かつ AC1 < 0.65 → **FAIL**（規約改訂が必要）

```json
  "after_reasoning": "改革により内部にビジョンの明確さが生まれ（離）、外部環境も成長を後押し（乾）"
}
```

### hexagram番号の算出

King Wen配列に基づく。lower × upper の組み合わせで一意に決まる。

## 注意事項

1. **state_labelに引きずられすぎない**: 「どん底・危機」→ 自動的に坎、ではない。テキスト全体から判断する
2. **内卦と外卦の独立性**: 内と外は別の観点。同じ八卦になりうるが、常にそうではない
3. **曖昧な場合はreasoningに明記**: 「坎と艮で迷ったが、○○の記述からXを選択」
4. **action_typeは直接の判定対象ではない**: before/afterの状態に間接的に反映される
5. **決定木は補助ツール**: 最終判断はテキスト証拠に基づく。機械的適用を避ける
6. **uncertainは最後の手段**: まず境界事例ルールと競合ルールで解決を試みる
