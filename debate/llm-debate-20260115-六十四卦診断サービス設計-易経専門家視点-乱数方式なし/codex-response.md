# Codex (OpenAI GPT-5.2) の批評

### 結論（最初に明言）
乱数方式を禁じた「六十四卦診断」は、**占筮ではなく分類（診断）である**。よって設計は **案B（構造化質問）を中核**に置き、**案A（類似事例検索）を根拠提示として従属させる**以外に成立しない。**案C（LLM候補提示+選択）を中核に据える設計は不合格**である。理由は「決定根拠が説明不能」「再現性が壊れる」「ユーザーの自己投影を増幅して診断価値が消える」の3点で致命的だからである。

---

### 批判的分析
- **論理的整合性**  
  - 「状況→卦を一意に判定する」主張は誤りである。なぜなら易経の解釈は**視点・目的・時間断面**で卦が変わる体系であり、入力文から“唯一の正解卦”が導出される前提が成立しない。  
  - よってサービスは「唯一解を当てる」ではなく、**視点を固定した分類規則を提示し、その規則に従って主卦・爻位を決める**設計でなければ論理が破綻する。
- **実現可能性（隠れコスト）**  
  - **案A単体**は不可能である。類似度は実装できても、**「似ている＝同じ卦」**の妥当性を担保できない。12,000件が多いことは根拠ではなく、**偏りの増幅装置**になる。  
  - **案B**は実装可能である。必要なのは「八卦（8分類）×内外×爻位（6段階）」の**明示的ルーブリック**であり、これは仕様として書ける。UXが重い問題は、質問数を削る設計努力の問題であり、方式の欠陥ではない。  
  - **案C**は運用コストが爆発する。モデル更新・プロンプト調整・安全対策で出力が揺れ、**同じ入力でも卦が変わる**。診断サービスとしての信用を自ら破壊する。
- **新規性の真偽**  
  - A/B/Cはいずれも新規ではない。Aは検索、Bはルールベース診断、Cは推薦UIである。新規性が成立するのは「易経の解釈学に沿ったルーブリック」と「12,000事例の根拠提示を結合した診断体験」だけである。
- **スケーラビリティ**  
  - Aはデータ増で検索は伸びるが、**ラベル品質（卦・爻の付与基準）が腐る**と全体が崩壊する。スケールするのは件数ではなく**基準の一貫性**である。  
  - Bはスケールする。ルーブリックが仕様化されていれば、ユーザー数が増えても判定は安定する。  
  - Cはスケールしない。説明責任と再現性の要求に耐えない。
- **反例・エッジケース**  
  - 視点が複数（当事者A/B、第三者、組織/個人）が混在する入力では、AもCも破綻する。Bは最初に**「誰の視点で診断するか」**を固定する質問を置けば崩れない。  
  - 状況が複数テーマ（仕事＋恋愛＋健康）に跨る入力で、A/Cは混線する。Bは**テーマ分割→主テーマ優先**の規則を持てば処理できる。  
  - 悪意入力（釣り、虚偽、過剰な比喩）ではCが最も脆い。Bは回答選択式なら耐える。
- **代替案との比較**  
  - 最適は「**B（決定論）＋A（根拠提示）＋必要最小限の自由記述**」である。自由記述は判定に使わず、解釈文の文脈補強に限定する。判定に使うと再現性が壊れる。

### Claudeの意見への反論/修正
- 「上卦=外、下卦=内」「爻=時間経過」という断定は**学習用の簡略化**であり、そのまま判定仕様にすると誤りである。易経は内外・時間だけでなく、**主客・進退・応比・位不位**などの構造で読む体系である。設計に使うなら「外/内」を**ユーザー入力の軸**として採用する、と明確に限定せよ。  
- 「状況→卦は本質的に難しい」という指摘は正しい。ただし結論が甘い。難しいのではなく、**一意判定を正解として売る発想が間違い**である。診断サービスとして成立させるには、最初に「本サービスは乱数占ではなく、定義済みルーブリックによる分類である」と**契約を固定**しなければならない。  
- 案Aの欠点を「類似度が曖昧」と述べているが弱い。曖昧ではなく、**類似検索は“卦の必然性”を一切保証しない**。A単体で卦を決める設計は、説明責任を果たせない時点で失格である。  
- 案Cの欠点を「不透明」とだけ言うのも甘い。Cは**再現性・検証可能性・信用の持続**の3要件を満たさない。中核方式として採用してはならない。

### より正確な見解
- 本質は「乱数を捨てた瞬間、易経は“占い”ではなく“構造診断フレームワーク”になる」ことである。ここを曖昧にすると、ユーザーは「当て物」を期待し、必ず不信感に到達する。  
- 方式設計は次で確定する：  
  1) **視点固定**（誰の立場で読むか）  
  2) **主テーマ固定**（何を問うか）  
  3) **内外・主客・能動/受動・硬/柔・停/動**などの軸を少数質問で確定し、上卦下卦を決める（B）  
  4) **進展段階（初〜上）を質問で確定**して爻位を決める（B）  
  5) その結果に対し、12,000事例から**同卦同爻（または同上卦/同下卦）**を引いて「なぜそう読むか」を提示する（A）  
- 無料/有料差は「結果」ではなく「付加価値」に限定せよ。無料は主卦・爻・要点・根拠事例数の提示まで、有料は根拠事例の詳細、分岐シナリオ、行動計画、継続ログ、対話深掘りを提供する。結果が動いた瞬間に信用は死ぬ。
