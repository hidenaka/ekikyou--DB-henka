# 議題コンテキスト: trigger/action抽出と六十四卦マッピング

## 現状

### データベース概要
- 総事例数: 12,871件
- pattern_type: 100%充填（14種類の変化パターン）
- story_summary: 100%充填（経緯の文章記述）
- outcome: 100%充填（Success/Failure等）

### 欠けているフィールド
- trigger: 0%（変化のきっかけ）
- action: 0%（対応行動）
- hexagram: 0%（六十四卦）
- trigram_from/to: 0%（変化前後の八卦）

### story_summaryの例
```
【日立製作所】Shock_Recovery → Success
story: リーマンショックの影響で国内製造業史上最大の7873億円の赤字を計上。
「沈む巨艦」と揶揄されたが、不採算事業の撤退と社会イノベーション事業への
集中を行い、奇跡的なV字回復を果たした。
```

## 提案

### 提案1: trigger/action抽出
story_summaryからLLMで以下を抽出:
- trigger: 変化のきっかけ（例: 「リーマンショック」「経営者交代」）
- action: 対応行動（例: 「不採算事業撤退」「事業集中」）

### 提案2: 六十四卦マッピング
事例を六十四卦にマッピング:
- 現在のpattern_typeを卦に対応させる
- または、trigger/action/outcomeの組み合わせから卦を決定

## 懸念点

1. **LLM抽出の品質**: 12,871件を自動抽出した場合の精度
2. **卦マッピングの恣意性**: 誰がどの基準で卦を割り当てるか
3. **既存pattern_typeとの整合性**: 14パターン vs 64卦の関係
4. **コスト**: 12,871件×LLM呼び出しのAPI費用と時間

## 目的
「どういうときにどのような状態になるか」の変化ロジックを構造化し、
傾向分析に使えるデータベースにすること。
