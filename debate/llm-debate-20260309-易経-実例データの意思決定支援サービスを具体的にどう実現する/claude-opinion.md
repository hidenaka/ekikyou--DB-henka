# Claude (Anthropic) の見解

実現に向けた具体的な設計案を提示する。

【MVP定義】
最小実行可能製品は「テキスト相談→レポート返送」の非同期型。ココナラの占い・人生相談カテゴリで開始するのが最も低コスト。理由：(1)占い需要が既に集まっている場所、(2)初期費用ゼロ、(3)テキストベースなので自動化しやすい。

【入力設計】
ユーザーの漠然とした不安をどう構造化するかが最大の技術課題。提案：段階的ヒアリング方式。
Step 1: 自由記述（今の状況と感じている違和感を書いてもらう）
Step 2: LLM（Claude API）で自由記述から以下を抽出：
  - 領域（キャリア/人間関係/生活/健康/お金）
  - before_state（停滞/成長痛/安定だが違和感/危機 等）
  - 行動傾向（攻めたい/守りたい/逃げたい/分からない）
Step 3: 抽出結果から確率テーブルで卦を割り当て

【出力設計（5点固定）】
1. 現在地：卦の象意 + 爻位による段階認識。古典の卦辞を現代語で。ここが占い的体験の核
2. 今やるな：その卦・爻位で典型的に失敗する行動パターン。事例DBから「同じ状態で失敗した事例」を引用
3. 今やれ：その卦・爻位で成功事例に多かった行動。事例DBから具体例を引用
4. 反対視点：錯卦・綜卦・互卦から導かれる別の見方。物語的に提示
5. 類似事例：DBから2-3件の実例サマリー（企業名・個人名は匿名化）

【象徴と事例の分離】
レポートを2パートに分ける：
Part A「易経の読み解き」→ 卦辞・爻辞ベースの象徴的解釈（占い的体験）
Part B「実例データからの参照」→ 11,336件からの統計的傾向と具体事例（データ的裏付け）
ユーザーには「Part Aは古典の知恵による読み解き、Part Bは実データの参照です」と明示

【価格】
ココナラの占い・人生相談の相場は3,000-10,000円。初期は5,000円で、レポート1件あたりの工数はLLM自動生成+人間レビューで30分以内を目指す。

【技術スタック】
入力処理: Claude API → 構造化抽出
卦割り当て: 既存の確率テーブル
事例検索: cases.jsonl全文検索 + 八卦フィルタ
レポート生成: Claude API → テンプレートベース
出力: PDF or Markdown

【課題】
- LLMによる卦割り当ての一貫性（同じ入力→同じ卦になるか）
- kappa=0.561の事例データの信頼性をどう補うか
- 一人運用でのスケーラビリティ限界
