# Codex (OpenAI GPT-5.4) の批評

### 結論（最初に明言）
この企画は**成立するが、Claude案のままではMVPにならない**。  
破綻点は明確である。**商品約束が矛盾していること、卦割り当てロジックが不安定であること、事例参照が擬似的な裏付けに堕ちていること、そしてチャネルと価格が運用現実に合っていないこと**である。  
実現すべきMVPは「占いの自動化」ではない。**構造化問診 + 粗い類型化 + 非象徴タグによるケース参照 + 人間レビュー**である。

### 批判的分析
- **論理的整合性が崩れている。** 合意済み方針は「予測しない。参照する」である。他方、既存サービス方針には「このまま進んだ場合に起こりやすい未来の流れを見せる」とある。これは予測である。Q6予測モデルを棄却した時点で、この約束は撤回すべきである。
- **入力設計が粗すぎる。** 自由記述から「領域」「before_state」「行動傾向」を抽出して卦を割り当てる設計は誤りである。64卦、さらに爻位まで含めるには情報量が足りない。同じ悩みでも言い回しで別卦になり、再現性が壊れる。
- **象徴と事例の分離がUI上だけで、推論上は分離されていない。** 同じ卦割り当てを使って事例を引くなら、象徴が事例選択を汚染し、その事例が象徴を正当化する循環が起きる。これは分離ではなく自己補強である。
- **kappa=0.561で「統計的傾向」や「成功事例に多い行動」を語るのは過剰である。** その精度では fine-grained な行動推奨の根拠として弱い。しかも企業・個人・家族・国家を同じ「成功/失敗」で扱うのは定義破綻である。
- **新規性の見立てが甘い。** このサービスは方法論として新しくない。ケースベース推論に象徴的フレーミングを被せた構成である。差別化要因は概念ではなく、11,336件の資産と編集品質である。
- **スケーラビリティが読み違えられている。** ボトルネックは生成ではない。曖昧入力の補足、危機ケースの判定、事例の妥当性確認、匿名化、修正依頼対応が詰まる。30分/件は成立しない。
- **エッジケースに無防備である。** 「不安はあるが決めることがない」「退職直前」「DV」「借金」「医療」「投資判断」では、断定的助言は危険である。拒否基準と退避導線がない設計は不適格である。
- **チャネル設計が雑である。** ココナラは需要がいる場所ではあるが、期待値を「当ててほしい占い」に寄せる。あなたが売りたいのは「参照による意思決定支援」である。入口と価値提供がズレる。
- **価格設計が甘い。** 5,000円で人間レビュー込み30分運用は、問い合わせ往復と品質保証を含めた瞬間に崩れる。安値で始めるほど、精度より慰撫を求める顧客が増え、設計思想が壊れる。

### Claudeの意見への反論/修正
- **「非同期テキスト相談→レポート返送」は正しい。** これはMVPとして妥当である。同期セッションは工数を爆発させる。
- **「ココナラ開始が最適」は誤りである。** ココナラは集客の近道であって、プロダクト検証の最適解ではない。マーケットプレイスの文脈が、商品の意味を占いに固定してしまう。
- **「自由記述→LLM抽出→確率テーブルで卦割り当て」は誤りである。** これは安定しないし、説明できない。診断サービスの中核に置くべきでない。
- **「Part A/Part Bに分ければ象徴と事例は分離できる」は誤りである。** 分けるべきは見せ方ではなく、検索経路と推論根拠である。
- **「失敗事例」「成功事例」で出す」は粗雑である。** 成功の定義が領域横断で揃っていない以上、その二分法は乱暴である。使うなら「過剰行動」「過少行動」「転換成功」のような行動型で整理すべきである。
- **「全文検索 + 八卦フィルタ」で十分」は甘い。** それでは表層一致しか拾えない。必要なのは、非象徴タグによる検索である。
- **「5,000円」は安すぎる。** その価格で守れるのは簡易診断だけである。カスタム事例参照付きレポートの価格ではない。

### より正確な見解
このサービスの本質は、**未来を当てることではなく、意思決定を曖昧さの中で前進可能にすること**である。したがって、MVPは次の形に絞るべきである。

- **商品定義**は「人生の現在地診断」ではなく、正確には「局面診断と次の一手の制約提示」である。未来予測は切る。
- **入力**は自由記述中心ではなく、まず「今、何を決めたいか」を一文で書かせる。これが書けないユーザーには本診断を売らない。次に「時間圧」「可逆性」「外部制約」「資源余力」「対人摩擦」「主体性」のような6軸を強制選択で取る。
- **分類**は64卦直行ではなく、まず8から16の変化パターンに粗く落とす。卦はその後に重ねる物語レイヤーであって、最初の分類器ではない。
- **事例検索**は卦ではなく、非象徴タグで行う。領域、局面、制約、意思決定の可逆性、利害関係者の衝突、残余資源で引く。卦は補助属性に留める。
- **出力**は5点固定でよい。ただし各項目に「根拠の種類」を明記するべきである。`現在地=象徴解釈`、`今やるな/今やれ=ケース参照 + 適用条件`、`反対視点=認知反転`、`類似事例=参照例` と分ける。
- **チャネル**は初手でLINEではない。初手は自前Webフォームである。入力制御、拒否基準、同意文、危機ケース分岐を持てるからである。ココナラは型が固まった後の集客チャネルとして使う。
- **価格**は二段階が正しい。簡易診断なら5,000円以下でも成立する。だが、事例選定と人間レビューを含む本命商品は8,000円から15,000円帯でなければ運用が持たない。
- **評価指標**は予測精度ではない。見るべきは、同一入力での再現性、ユーザーの納得度、7日以内の行動着手率、修正依頼率、危機ケースの誤通過率である。

要するに、**Claude案は「それっぽい体験」を先に作りすぎており、診断サービスに必要な安定性・説明可能性・運用現実を後回しにしている**。  
成功条件は逆である。**まず安定した問診と粗い類型化を作り、その上に易経の象徴を載せること**である。これが唯一、商品としても運用としても筋が通る。
