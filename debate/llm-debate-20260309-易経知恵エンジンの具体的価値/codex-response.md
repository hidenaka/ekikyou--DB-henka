# Codex (OpenAI GPT-5.4) の批評

### 結論（最初に明言）
易経知恵エンジンの価値は、「知恵を与えること」ではない。価値は「曖昧で高リスクな状況を、型・禁忌・視点転換に分解し、誤った一手を止めること」である。  
したがって、広い意味でのB2C「知恵サービス」としては弱い。成立するのは、人生や経営の分岐点で使う単発の意思決定レビュー、または専門家向けの構造化支援ツールである。

### 批判的分析
- 価値仮説の中心がずれている。ユーザーは「知恵」そのものに金を払わない。金を払うのは、「判断ミスの下振れを減らしたい」「頭の混乱を短時間で整理したい」「今やると危険な一手を避けたい」という効用に対してである。
- 「名前がつく価値」は必要条件であって十分条件ではない。ラベル付けは友人、書籍、コーチ、ChatGPTでもできる。ラベルだけでは支払い理由にならない。
- 「予測しない」は正しい。その代わり、「結果が良くなる」と匂わせた瞬間に論理が崩れる。このサービスが売るべきものは成果保証ではなく、判断プロセスの改善である。
- データ基盤の説明に重大な不整合がある。総数は11,336件と書かれているが、内訳は5,510+3,217+1,381+787+2,165=13,060件で一致しない。ここが崩れている状態で「DB資産」「根拠」を売りにすると、価値以前に信用を失う。
- 11,336件という件数自体にも意味はない。未構造の逸話集は知恵ではなく雑音である。価値になるのは、各事例が「状況の型」「段階」「取った行動」「禁忌」「転機の兆候」「結果の分岐」に正規化されている場合だけである。
- 64卦×6爻の体系は強みである一方、現実を単純化しすぎる危険もある。現実の問題は複数当事者、複数時間軸、表面課題と構造課題のズレを含む。単一の卦・爻に押し込む設計は誤診を生む。単一診断ではなく、複数候補と分岐条件が必須である。
- 新規性も誇張されている。視点変換、内省支援、禁忌提示、事例参照は既存手法でもできる。新しいのは「易経の分類体系で一貫してそれを行うこと」だけである。ここに再現性と説明責任がなければ、新規性は消える。
- スケーラビリティはB2C月額モデルと相性が悪い。人生の岐路は頻度が低い。継続価値が成立するのは、前回判断の追跡、兆候の観測、見立ての更新まで持つ場合だけである。単なる「月1チェック」は続かない。
- 代替比較も甘い。占いには「断定してくれない」点で負ける。コーチングには「伴走と文脈理解」で負ける。ChatGPTには「一貫した分類と根拠提示」を示せなければ負ける。友人相談には「事情把握と信頼関係」で負ける。勝てるのは、感情に巻き込まれず、短時間で、構造化された禁忌と盲点を出す場面だけである。
- エッジケースも重い。自己正当化された入力には弱い。医療・法務・投資のような高リスク領域では誤用が危険である。複数当事者が対立する局面では、自分視点の入力だけで診断すると誤導が起きる。

### Claudeの意見への反論/修正
- 「名前がつく価値」は正しいが、主価値ではない。名前は入口にすぎない。支払いを生むのは、その名前が「何を止めるべきか」「何をまだ決めるべきでないか」に接続された時だけである。
- 「3000年かけて全パターンに名前をつけた」は誇大である。64卦384爻は強い分類語彙だが、現実の全パターンを被覆するとは言えない。ここを全能視すると設計が壊れる。
- 「やるな」の価値はClaudeの指摘の中で最も強い。ここは正しい。ただし、禁忌は現在地の特定とセットでなければ機能しない。診断が曖昧なら禁忌も曖昧になる。
- 「ChatGPTより優れる」は自動では成立しない。勝つ条件は三つだけである。出力が一貫していること、なぜその見立てなのか説明できること、類似事例と古典根拠を並べて示せること。この三つがなければ、ChatGPTに易経プロンプトを入れたものと差がない。
- 「体系的で根拠があり安い」という評価は甘い。「体系的」は作れるが、「根拠がある」は言い過ぎである。あるのは因果根拠ではなく、解釈根拠と事例根拠である。この差を曖昧にすると危険である。
- 「月1回の現在地チェック」は弱い。継続価値は、前回の仮説、実際に起きた変化、見逃していた兆候を比較できる時にしか生まれない。毎月ラベルを付けるだけでは飽きる。
- 「無くなっても困らない人が多い」は正しい。そしてそれは軽い問題ではない。これは日常必需品ではない。だからこそ、広い“知恵サービス”ではなく、狭く深い“意思決定レビュー”として設計すべきである。

### より正確な見解
- このエンジンの本当のJTBDは、「正解を教えてほしい」ではない。「いま自分は何を見誤っているのか」「今打つと傷が深くなる一手は何か」「判断前に観測すべき兆候は何か」を短時間で整理したい、である。
- 勝てる場面は限定される。転職、共同創業、撤退判断、組織再編、離別、対人膠着のように、感情が強く、利害が絡み、拙速な一手のコストが大きい場面である。ここでは「当てる」より「踏んではいけない地雷を示す」方が価値になる。
- ユーザーが金を払う瞬間は、「一般論ではなく、自分の状況に即した禁忌・盲点・反対視点が5分から15分で得られる」と体感した時である。抽象的な知恵では払わない。具体的な判断停止線と観測項目が出た時に払う。
- 1回利用の価値は強い。現在地の候補、禁忌、優先順位、反対視点、転機シグナル、類似事例が手に入るからである。継続利用の価値はそのままでは弱い。継続価値を作るには、前回判断、実行した行動、現れた兆候、次の変化を時系列で検証する仕組みが必要である。
- 「知恵」はそのままでは売れない。売れるのは、知恵を意思決定の作業手順に変えた時だけである。つまり、「診断」「禁忌」「観測」「再評価」のワークフローに落とし込んだ時だけ商品になる。
- DBが知恵に変わるメカニズムは、事例を見せることではない。事例から反復する失敗パターン、段階ごとの禁忌、転機の兆候、視点のズレを抽出し、相談に対して「なぜこの型なのか」「どこが一致し、どこが一致しないのか」を示すことである。DBの役割は引用集ではなく、反証付きの類推装置である。
- 不可欠性は個人向けでは弱い。無くなって困る人は少ない。不可欠になり得るのは、意思決定支援を仕事にする専門家、または重大な岐路で同じ誤りを繰り返す反復ユーザーである。最も筋が良い戦略は、汎用B2Cではなく、起業・組織変化・キャリア移行のように型が見えやすい領域へ絞り、個人向け「知恵エンジン」ではなく「意思決定レビューエンジン」として売ることである。
