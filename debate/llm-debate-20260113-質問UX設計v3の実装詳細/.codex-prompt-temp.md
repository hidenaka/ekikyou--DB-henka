# HaQei診断 質問UX設計 v3

## 背景

LLMディベート（2026-01-13）の結論に基づく設計。現行の質問が「データベースの分類軸に寄りすぎ」で答えにくい問題を解決する。

## 設計原則

1. **内部軸は維持** - Q1〜Q5の分類ロジックは有効、問題は「写像」
2. **行動指標への翻訳** - 抽象概念を具体的行動・状況で測定
3. **混合状態の許容** - 単一選択ではなく複数選択・段階評価
4. **設問バンク方式** - 動的生成ではなく事前設計の設問から適応出題
5. **説明可能性** - 「なぜこの結果か」を明示

## 設問バンク設計

### Q1: 変化の性質（changeNature）

**現行**: 「変化の性質は？」→ 拡大/収縮/維持/転換（直接選択）

**新設計**: 行動指標で間接測定

| ID | 設問文 | 測定対象 | 回答形式 |
|----|--------|----------|----------|
| CN1 | 今、新しい役割・ポジション・責任を引き受けようとしていますか？ | 拡大 | 5段階（全くない〜まさにそう） |
| CN2 | 今の仕事量や責任を減らしたい、手放したいと感じていますか？ | 収縮 | 5段階 |
| CN3 | 今の状態を安定させる・守ることが最優先ですか？ | 維持 | 5段階 |
| CN4 | 業界・職種・働き方そのものを変えることを考えていますか？ | 転換 | 5段階 |

**スコアリング**: 各回答の最大値を主軸、2番目を副軸として記録

### Q2: 主体性（agency）

**現行**: 「主体性は？」→ 自ら動く/受け止める/待つ（直接選択）

**新設計**: コントロール感の段階尺度

| ID | 設問文 | 回答形式 |
|----|--------|----------|
| AG1 | 今起きている（または起きそうな）変化について、自分でどの程度コントロールできると感じますか？ | 5段階（全くできない〜完全にできる） |
| AG2 | この状況を変えるために、あなたから動き出す予定はありますか？ | 5段階（待つしかない〜すでに動いている） |

**スコアリング**:
- 平均4-5 → 「自ら動く」
- 平均2-3 → 「受け止める」
- 平均1-2 → 「待つ」

### Q3: 時間軸（timeframe）

**現行**: 「時間軸は？」→ 今すぐ/数ヶ月/1年以上（直接選択）

**新設計**: 具体的な期限で測定

| ID | 設問文 | 回答形式 |
|----|--------|----------|
| TF1 | この変化はいつ頃起きる（または起こす）予定ですか？ | 選択式: 1ヶ月以内/3ヶ月以内/半年以内/1年以内/1年以上先/わからない |
| TF2 | 決断や行動のタイムリミットはありますか？ | 5段階（全くない〜差し迫っている） |

### Q4: 関係性（relationship）

**現行**: 「関係性は？」→ 個人/組織内/対外（直接選択）

**新設計**: 影響範囲で測定（複数選択可）

| ID | 設問文 | 回答形式 |
|----|--------|----------|
| RL1 | この変化は主に誰に影響しますか？（複数選択可） | ☐ 自分自身 ☐ 家族 ☐ 同僚・チーム ☐ 組織全体 ☐ 顧客・取引先 ☐ 業界・社会 |
| RL2 | 意思決定に関わる人は何人くらいですか？ | 選択式: 自分だけ/2-3人/4-10人/10人以上 |

**スコアリング**:
- 「自分自身」「家族」のみ → 個人
- 「同僚・チーム」「組織全体」含む → 組織内
- 「顧客・取引先」「業界・社会」含む → 対外

### Q5: 感情基調（emotionalTone）

**現行**: 「感情は？」→ 前向き/慎重/不安/楽観（単一選択）

**新設計**: 複数感情の強度測定（複数選択可）

| ID | 設問文 | 回答形式 |
|----|--------|----------|
| ET1 | 今の状況について、以下の感情をどの程度感じていますか？ | 各感情を5段階で評価 |
| | - ワクワク・期待感 | 5段階 |
| | - 慎重さ・用心深さ | 5段階 |
| | - 不安・心配 | 5段階 |
| | - 楽観・なんとかなる感 | 5段階 |

**スコアリング**: 最も高い感情を主軸、2番目を副軸

## 適応出題ロジック

### Phase 1: コア質問（必須2問）
1. **CN1-CN4の統合版**: 「今、あなたの状況に最も近いのはどれですか？」
   - A) 新しいことを始める・広げる段階
   - B) 手放す・整理する段階
   - C) 今の状態を守る・安定させる段階
   - D) 方向転換・大きく変える段階
   - ※複数選択可、「最も近い」+「次に近い」

2. **AG1**: コントロール感（5段階）

### Phase 2: 判別不足軸の追加質問
- 信頼度 < 70% の軸のみ追加質問
- 各軸から1問ずつ（最大3問追加）

### 質問数の期待値
- 最短: 2問（Phase 1で高判別）
- 平均: 4問
- 最長: 5問

## Claudeの意見（参考）
設計v3の実装方針: (1)設問バンクは各軸3-5問で計15-20問を事前設計、スコアリングは加重平均で混合状態を数値化 (2)適応出題は信頼度70%閾値で判定、Phase1の2問で高判別→不足軸のみ追加 (3)UIは段階尺度をスライダーではなくボタン5つで実装（モバイル操作性重視）(4)「なぜこの結果か」は回答値と判定ロジックの対応をテンプレート化して自動生成 (5)KPIは自己納得度を結果画面に5段階で埋め込み、再テスト一致はユーザーIDで追跡。懸念点: 設問バンクの質問文が本当にユーザーに分かりやすいか、5段階評価の中央値偏り（3に集中）をどう防ぐか。

## 評価の観点（必ずすべて検討すること）
1. **論理的整合性**: 主張と根拠に矛盾はないか？前提は妥当か？
2. **実現可能性**: 技術的・実務的に本当に実装できるか？隠れたコストは？
3. **新規性の真偽**: 本当に新しいのか？既存手法との差分は明確か？
4. **スケーラビリティ**: 規模が大きくなっても成立するか？
5. **反例・エッジケース**: この主張が破綻するケースは？
6. **代替案との比較**: より優れたアプローチは存在しないか？
