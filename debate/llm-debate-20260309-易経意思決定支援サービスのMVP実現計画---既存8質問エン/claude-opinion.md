# Claude (Anthropic) の見解

GPT-5.4の方針に同意した上で、具体的な実現計画を立案する。

【前提：やること/やらないこと】
やること：
- 入口を自由記述に変更（今何を決めたいか）
- 8質問を補助確認に下げる
- 出力を5点固定に再編（現在地/今やるな/今やれ/反対視点/類似事例）
- 象徴解釈と事例参照を分離表示
- 成功率表示の排除
やらないこと：
- 6軸への全面再設計
- 統計テーブルの作り直し
- 事例の再ラベル化

【既存資産の棚卸し】
完成済み：
- diagnostic_engine.py（8質問→八卦+勢+時+推奨行動）
- case_search.py（11,336件の条件付き分布+類似事例検索）
- feedback_engine.py（5層レポート生成）

- iching_texts_ctext_legge_ja.json（64卦の日本語訳）
- statistics_table.json（条件付き統計）
- 336テスト

【実現計画案】

Phase 0: 出力フォーマット再設計（1-2日）
- 既存5層レポート → 5点固定フォーマットへのマッピング定義
  Layer1（現在地の読み解き）→ 「あなたの現在地」
  Layer2（変化方向）→ 削除または「今やれ」に統合
  Layer3（隠れた力学）→ 「反対視点」
  Layer4（事例分布）→ 「類似事例」
  Layer5（問いかけ）→ 削除
- 「今やるな」「今やれ」は既存の推奨行動スコアから導出
- 象徴パート（卦辞・爻辞ベース）と事例パート（統計・事例）を明示的に分離

Phase 1: 入口の変更（2-3日）
- 自由記述入力フォーム追加（「今、何に迷っていますか？」）
- Claude APIで自由記述 → 8質問の回答を自動推定
- 推定結果をユーザーに確認させる（曖昧な部分だけ質問）
- 既存diagnostic_engineにそのまま流す

Phase 2: 出力テンプレート実装（2-3日）
- 5点固定のMarkdown/HTMLテンプレート作成
- Part A（易経の読み解き）: 卦辞・爻辞の現代語訳 + 象意解説
- Part B（実例からの参照）: 条件付き分布 + 類似事例2-3件
- 「成功率」→「N件中M件がこの結果」の頻度表示に変更
- 警告表示（危機ケースの退避導線）

Phase 3: MVP統合テスト（1-2日）
- エンドツーエンドテスト（自由記述→レポート生成）
- 5人程度のテストユーザーで試行
- レポートの読みやすさ・納得度を確認

Phase 4: チャネル準備（1-2日）
- 自前Webフォーム（Vercel等で無料デプロイ）
- 簡易LP（サービス説明+入力フォーム）
- 利用規約（予測でないこと、医療相談でないことの明示）

Phase 5: β公開（随時）
- 無料 or 1,000円で10-20件の実例を集める
- 納得度・行動着手率を測定
- フィードバックで出力を改善

合計：約2週間でMVPリリース可能

【技術的な具体策】
- 自由記述→8質問変換：Claude APIのstructured outputで8質問の回答形式に変換するプロンプトを作る
- 出力から成功率を消す：statistics_tableの数値は内部スコアリングにのみ使用、ユーザーには頻度（N件中M件）で見せる
- 象徴と事例の分離：テンプレートで物理的にセクション分け

【リスクと対策】
- 自由記述→8質問変換の精度 → テストで検証、低精度なら確認質問を増やす
- 危機ケースの誤通過 → キーワード検出で自動退避（医療/法務/DV等）
- Claude APIコスト → 1件あたり約50-100円（入力抽出+レポート生成）
