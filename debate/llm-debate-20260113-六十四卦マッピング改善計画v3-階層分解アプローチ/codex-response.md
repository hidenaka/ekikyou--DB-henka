# Codex (OpenAI GPT-5.2) の批評

### 結論（最初に明言）
この改善計画v3は「64クラス直分類を捨てて階層化する」という方向だけは正しいが、**ラベル定義・評価設計・アノテーション設計が破綻しているため、このまま実行しても再現性のある64卦マッピングは成立しない**。

### 批判的分析
- **論理的整合性（破綻点）**  
  - 「下卦=before、上卦=transformation」という割当は恣意である。ビジネス記述は“状況→行動”に綺麗に分割されず、同一文に混在する。したがってこの前提に依存したラベリングは体系ではなく作文になる。  
  - 64卦は上卦×下卦の**決定関数**である。なのに「上/下の精度70%」と「64卦Top-3精度80%」を同列の成功基準に置くのは設計として不整合である（組合せ誤差の扱いが未定義）。
- **実現可能性（コスト見積が誤り）**  
  - κを語るなら最低でも二重アノテーション+調停が要る。**500件を“ゴールド”と呼びつつ単独ラベラー前提の工数（40–60h）を置くのは誤りである。**二重化と調停で工数は構造的に倍化する。  
  - 50件パイロットでκ≥0.6は統計的に不安定で、合否判定として雑である。ここで「OK」と誤判定した瞬間に以後の全工程が汚染される。
- **ラベルの操作的定義（最重要欠陥）**  
  - 提示された八卦キーワードは**互いに排反でなく**、弁別不能である。例：「革新/攻勢」は乾・震・離に重なり、「危機/困難」は震・坎に重なる。「適応/浸透」は巽・坤・兌に滑る。これではκが上がらない。  
  - 「unknown」を真ラベルとして置くなら、**unknownの適用条件を“必要十分”に定義**しなければならない。現状は逃げ道として機能し、データ品質を崩壊させる。
- **unknown と abstain の分離（発想は正しいが未完成）**  
  - abstain率≤10%は無意味である。重要なのは**棄却したときに誤りをどれだけ減らしたか（選択的分類のリスク-カバレッジ）**であり、率だけ制約しても品質は上がらない。  
  - LLMの自己申告で棄却理由を出す設計は信頼できない。校正（しきい値設計）が欠落している。
- **新規性の真偽**  
  - 階層分類、検索+LLM、Top-k評価、unknown/abstain分離は既存の定石である。新規性は「易経ラベルでビジネス事例を構造化する応用」に限定され、方法論の新規性ではない。
- **スケーラビリティ（成立条件が欠ける）**  
  - キーワード駆動の定義はドメイン・文体・言語が変わると即死する。スケールさせるなら、キーワードではなく**判定軸（観測可能な特徴）と優先規則**が必要である。  
  - 検索ベースはデータ重複・類似文面に弱い。分割戦略（同一ソース・同一企業・同一テンプレの分離）が無い評価は数値が虚偽になる。
- **反例・エッジケース（計画が必ず破綻する例）**  
  - 危機対応でリスク管理しつつ強行突破する案件：震・坎・乾が同時に立つ。現行定義ではラベラーが割れる。  
  - 公開PRで提携交渉も進める案件：離と兌が衝突する。  
  - 「停止して撤退線を引く」案件：艮と坎が衝突する。  
  - 文章が結果報告と次アクションを混在：stageが定まらずearly/mid/lateが崩れる。  
  - 情報欠落（行動が書かれていない/背景が書かれていない）：unknownの乱発か強制ラベルで破綻する。

### Claudeの意見への反論/修正
- **正しい点**  
  - 64直分類の廃止と階層分解は正しい。ここだけがv3の実質的前進である。  
  - unknown（データ側）と abstain（モデル側）の分離も正しい。評価の混線を防ぐ。
- **誤り・甘さ（修正必須）**  
  - 八卦の定義をキーワード列挙で済ませたのは誤りである。**排反性・優先規則・反例**が無い定義は再現性を生まない。  
  - 成功基準が不整合である（上/下精度と64卦Top-3の関係が未定義）。まず評価指標を数学的に整合させよ。  
  - 工数見積が現実を無視している。二重アノテーションと調停を入れた瞬間に崩れる。  
  - フォールバックで「8卦を4グループに統合」は、64卦マッピングという目的自体を破壊する。撤退案として出すなら「64を捨てる」と明言し、成果物の定義を作り直せ。

### より正確な見解
- このプロジェクトのボトルネックはモデルではない。**ラベル体系の弁別可能性（＝人間が一致して付けられる定義）**である。ここを解決しない限り、検索+LLMにしても数値は虚飾になる。  
- 修正するなら、順序は以下で固定せよ。  
  1) **上卦/下卦の意味割当を再定義**（例：下=内的資源/初期条件、上=外的環境/作用結果など。before/actionのような文章依存を捨てる）  
  2) **八卦を“判定軸+優先規則+対立ペアの判別規則”で定義**（乾vs震、坎vs艮、離vs兌等の衝突を先に潰す）  
  3) unknownを「情報欠落」「同点」「範囲外」で分割し、**適用条件を断言形で規定**する（逃げ道を禁止する）  
  4) κはコンポーネント別（上卦/下卦/stage）に測り、パイロットは**最低100件**で合否判定する  
  5) abstainは率ではなく**カバレッジ90%時の精度**など、選択的分類の指標で縛る  
- 64卦に固執するなら、最初から「64を当てる」ではなく、**上卦Top-k×下卦Top-kから生成される候補集合の再現率**を主指標に置け。これが階層分解の実用的ゴールである。
