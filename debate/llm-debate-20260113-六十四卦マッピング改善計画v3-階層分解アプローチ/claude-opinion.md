# Claude (Anthropic) の見解

## 改善計画v3の概要

Codex批評v2を全面的に受け入れ、**階層分解 + 検索ベース + 実用的な粒度**を原則とした計画を策定。

### 主要な設計変更

1. **64クラス直接分類を廃止**: 上卦(8)×下卦(8)の階層分解
2. **爻6段階を3段階に簡素化**: early(1-2)/mid(3-4)/late(5-6)
3. **unknown（真ラベル）とabstain（モデル棄却）を分離**
4. **検索+LLMのハイブリッド分類器**

### Step 1: 出力仕様の確定
- upper_trigram: 上卦（乾/坤/震/巽/坎/離/艮/兌/unknown）
- lower_trigram: 下卦（同上）
- hexagram: 合成結果（1-64）
- stage: early/mid/late/unknown
- model_action: abstain（モデル棄却）とその理由

### Step 2: 八卦の操作的定義
beforeの状況記述 → 下卦を推定
transformationの行動記述 → 上卦を推定
各八卦に判定キーワード例を明示

例:
- 乾: 創業、革新、攻勢、リーダーシップ
- 坤: 受容、忍耐、フォロワー戦略
- 震: 危機、衝撃、迅速対応
- 巽: 浸透、適応、柔軟対応
- 坎: 困難、リスク管理、撤退
- 離: 公開、ブランディング、可視化
- 艮: 停止、見直し、内省
- 兌: 交渉、連携、顧客対応

### Step 3: 段階の3段階定義
- early (爻1-2): 兆し・準備段階
- mid (爻3-4): 実行・転換段階
- late (爻5-6): 成熟・終末段階

### Step 4: アノテーション設計
Phase 4-A: パイロット50件（κ≥0.6目標）
Phase 4-B: ゴールドセット500件（64卦×8件程度）
- 訓練60%/検証20%/テスト20%分割

### Step 5: 分類器の構築
アプローチ: 検索+LLMハイブリッド
1. 類似事例検索（Top-5）
2. LLMにFew-shot分類させる

成功基準:
- 上卦/下卦 Top-1精度 ≥ 70%
- 64卦 Top-3精度 ≥ 80%
- 段階 Top-1精度 ≥ 75%
- 棄却率 ≤ 10%

### 工数見積もり
- Steps 1-3: 8-12時間
- Step 4-A: 8-12時間
- Step 4-B: 40-60時間（最大ボトルネック）
- Step 5: 16-24時間
- 合計: 72-108時間

### v2からの改善点
- 64クラス直接 → 階層分解（再現性向上）
- 6段階爻 → 3段階（判定容易化）
- unknown単一 → unknown+abstain分離（評価の正確化）
- 200件 → 500件（統計的信頼性）
- Top-1のみ → Top-1+Top-3（実用性）

### フォールバック
- κ≤0.5: 8卦を4グループに統合
- Top-3精度≤60%: Top-5に拡大
- 全体破綻: 八卦レベル（8クラス）に撤退

### 易経意味論の位置づけ
分類器の出力は「ビジネス事例の構造化ラベル」
易経の卦辞・爻辞は「ラベルに対する解釈・助言」
両者の整合性は後続の別プロジェクトで検証
