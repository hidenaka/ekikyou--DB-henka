# Claude統合分析

## 受け入れる批判

| 指摘 | 妥当性 | 対応 |
|------|--------|------|
| Gold件数目標は品質指標として不適切 | **同意** | KPIを精度ベースに再定義 |
| ドメイン≠信頼性（同一ドメイン内変動） | **同意** | ドキュメント単位評価への移行 |
| tier4_corporateの利益相反問題 | **同意** | COI軸を分離 |
| rejected URL補完は品質偽装リスク | **部分同意** | 監査付きで実施 |
| 正規表現111本は運用負債 | **同意** | 削減とメタデータ化 |

## 反論・留保

| 指摘 | 反論 |
|------|------|
| 分類設計を変えずに偏りを解けない | 分類と収集は独立に変更可能 |
| success_levelが循環参照 | outcomeから算出、ドメイン判定とは独立 |
| 新規性がない | 新規性は目標ではない。実用性が目標 |

## 採用する代替案（Codex提案）

### Phase A: 監査基盤構築
- Goldサンプル100件を無作為抽出
- 不適合タイプ（一次/二次/意見、COI、更新切れ）をラベル化
- KPI: Gold precision ≥ 90%

### Phase B: 二軸化
- tier4_corporateを「事実主張OK / 評価主張要裏取り」に分離
- COIフラグを新設

### Phase C: 統計改善
- 階層ベイズ（部分プーリング）でn不足解消
- n=10フラグ廃止、信用区間下限でゲート

### Phase D: 収集制御
- 言語別クオータ設定（日本60%/米国20%/欧州10%/その他10%）
- 非日本語シードリスト作成

## 結論

**現行アプローチは「可観測性の改善」としては成功だが、「品質の改善」としては設計に欠陥がある。**

件数目標を追う限り、品質を犠牲にする誘惑から逃れられない。KPIを精度ベースに再定義し、監査ベースの品質管理に移行すべき。
