# Codex (OpenAI GPT-5.2) の批評

### 結論（最初に明言）
Phase3v2の「同型性検証」は妥当ではない。**Test A/BがQ6の核心仮説（1ビット隣接＝局所遷移）を決定的に否定しており**、Test C/D/Eのpro結果は「Q6に固有な対応」を示さず、**周辺分布・重みスケール・(Z2)^6の一般対称性**で再現できる。したがって **「Moderate structural correspondence」という結論は誤りである。**

---

### 批判的分析

**1) v1.0批判への“形式的”対応は達成、しかし“本質”は未解決**
- 定義の明確化（G_obs vs Q6の構造類似）は前進である。一方で、**同型性という語を使うならエッジ対応が中核**であり、Test A/Bで崩壊している以上「同型性」の看板が成立しない。
- MCA変数の排除、64卦での直接検証、ラプラシアン同士の比較は改善である。しかし、**改善は「検定の体裁」を整えただけで、モデル仮説（Q6が遷移を規定する）の真偽を救っていない。**

**2) Test A/Bは“反証”、Test C/D/Eは“別物”を測っている**
- **Test A（エッジ重複）**：10/194しかQ6隣接に乗らない時点で、Q6の局所隣接遷移は破綻している。これは「一致しない」ではなく**「主要仮説が偽である」**を意味する。
- **Test B（ハミング距離）**：距離1が12/9228でほぼゼロ、距離2と4が支配的、距離6も10.8%。この分布は「Q6（距離1偏重）」と両立しない。  
  さらに、**偶数距離が98.6%**（0/2/4/6に集中）という事実は、Q6隣接ではなく**“偶数パリティ制約のあるXOR型操作”**を示す。ここが本質である。
- **Test C（スペクトル）**：スペクトル距離が小さいことは、**エッジ対応を意味しない**。スペクトルは非同型でも一致・近接し得る（cospectral問題）。加えて、重み付きラプラシアンの**スケール正規化が不明**で、未正規化なら「重み総量の合わせ込み」で距離が動く。ここを確定せずに“構造一致”を主張するのは不適切である。
- **Test D（錯卦対称性）**：距離6（全反転）が多いデータで、錯卦（ビット反転）対称性が出るのは自然であり、Q6隣接の支持ではない。しかも**32ペア中17ペアのみ有効**で、全体一般化は不可能である。
- **Test E（コミュニティNMI）**：Louvainは解が不安定で、コミュニティ数も16 vs 5で乖離している。NMI 0.276は“弱い一致”であり、**有意性はアルゴリズムと帰無の作り方で簡単に動く**。これを根拠に“対応”と断言するのは過大評価である。

**3) 帰無分布（afterシャッフル）の設計がテストごとに不整合**
- afterシャッフルは「周辺分布保存」の帰無としては一理あるが、**グラフの疎密・重み分布・次数（strength）構造を保存しない**。  
  スペクトルとコミュニティは次数・重みの影響を強く受けるため、ここを保存しない帰無は「Q6に似るか」ではなく「周辺分布が作る構造に似るか」を測る。
- Test Aでユニーク無向エッジ（194本）に落としている時点で、シャッフル後グラフのエッジ数や閾値処理が一致していなければp値は無効である。**同じ生成・同じフィルタを全条件に適用して初めて比較になる。**

**4) Fisher結合 p < 1e-300 は統計的に無効**
- Fisher結合は**p値の独立性**を前提にする。A〜Eは同一データ・同一シャッフル族・同一グラフ派生量であり独立でない。よって **Fisherのχ²較正は成立しない。**
- さらに致命的なのは、表の整合性である。**Test Aがz=-3.99なのにp=1.000**ということは、pを“pro方向の片側”で定義して反証（anti）をゼロ寄与にしている。これは結合検定として不正である。**反証を統合から除外した時点で、総合判定は結論ありきの集計である。**
- p=0表記も不正である。有限回の置換でp=0は定義できない（最低でも1/(B+1)）。**0を入れたFisherは統計量を任意に肥大させる。**

**5) Test Dの有効ペア17/32は「十分」ではない**
- 半数欠落は単なる検出力不足ではない。**選抜バイアス**である。出次数0のノードは「データがない」のではなく「その領域の構造を評価できない」ことを意味する。  
  結論は「17ペアで有意」までで止めるべきであり、「錯卦対称性がある」と一般化するのは誤りである。
- 回避策は存在する：擬似カウントでプロファイルを平滑化し、全ペアで比較できる指標（JS divergence等）に置換する。これをしない限り、Dは参考値である。

**6) 新規性・実用性・スケーラビリティの評価**
- 新規性は手法ではなく適用対象にある。グラフ比較（重複、距離分布、スペクトル距離、コミュニティ一致、対称性検定）は既存の定番である。新規と主張するのは誤りである。
- スケーラビリティは64ノードなので計算的問題はない。問題は**推論の頑健性**であり、帰無設計・前処理・指標選択が変わると結論が動く構造になっている。

### Claudeの意見への反論/修正
- 「ミクロ（隣接）とマクロ（スペクトル・対称性）が分離している」という整理は正しい。ここは認める。
- しかし、Claudeは**“マクロ構造が反映”という言い方でQ6の支持に寄せすぎ**である。Test C/D/EはQ6隣接の支持ではなく、**6bit空間一般の対称性・周辺分布が作る粗い構造**でも出る。
- 「Fisher p<0.001でModerate」は誤りである。前述の通り、**Fisherは前提崩壊**かつ**anti証拠を無視する集計**になっている。p値で結論を強化する態度自体が不適切である。
- 「large効果」という表現も甘い。例えばTest Cは差が0.025で、**実務的差分は小さい**。zが大きいことと意味が大きいことは同値ではない。

### より正確な見解
1. **Q6（距離1隣接）を“遷移モデル”として採用する判断は棄却である。**  
   根拠はTest A/Bで十分である。距離1遷移がほぼゼロである以上、Q6隣接を前提にした予測・制約・説明は誤作動する。

2. pro結果が示しているのは「Q6」ではなく、**(Z2)^6上の操作（XOR差分）に関する構造**である。  
   次にやるべき解析はグラフ比較ではない。**Δ = before ⊕ after（6bit差分）**の分布を一次データとして扱い、  
   - Δのハミング重み（0〜6）の分布  
   - Δが偶数重みに集中する事実  
   - 特定のΔ（全反転＝錯卦）が過剰に出る事実  
   をモデル化して検定することである。ここはQ6より一段深い本質である。

3. 結論文言はこう書き換えるべきである：  
   **「Q6隣接遷移仮説は反証された。一方で、6bit表現に基づく偶数パリティの対称性（および錯卦関連）を反映した粗い構造が観測遷移に存在する」**  
   これが現状の唯一の厳密な要約である。

4. 実用的価値は「予測」ではなく「表現・正則化」に限定される。  
   Q6近傍でスムージングする設計は誤りである。使うなら、Δ分布に整合した距離カーネル（距離2/4/6を主成分にする等）を学習して、**モデル選択と外部検証（ホールドアウトでの対数尤度・予測精度）**で価値を示すべきである。
