# LLM Debate: 六十四卦マッピング新アプローチ-本質八卦→15卦選択

## 議題
六十四卦マッピング新アプローチ-本質八卦→15卦選択

## Claude (Anthropic) の見解
## 新アプローチの提案

従来の設計（上卦×下卦→64卦）を根本から変更し、**本質の八卦を先に決定→その八卦が関わる15卦から選択**という2段階方式を提案。

### 設計思想

**現行設計の問題**:
- 上卦(8)×下卦(8)=64を一度に決定しようとする
- 抽象軸（時制/主体性など）と八卦の意味論の接続が恣意的
- 判定が困難で再現性が低い

**新アプローチ**:
1. **第1段階**: この事例の「本質」はどの八卦か？（8クラス分類）
2. **第2段階**: その八卦が上卦/下卦として関わる15卦から選択
3. **第3段階**: 64卦が決まってから、その卦の文脈で爻を判定

### 15卦の構造

各八卦は:
- 上卦として8卦に登場
- 下卦として8卦に登場
- 純卦（上下同じ）は重複 → 8+8-1 = **15卦**

例: 「震」が本質の場合
- 震為雷(51)、雷天大壮(34)、雷地予(16)、雷水解(40)...（震が上卦）
- 天雷无妄(25)、地雷復(24)、水雷屯(3)、火雷噬嗑(21)...（震が下卦）

### 第1段階: 本質の八卦を決定

**判定基準**: 事例の変化・行動の「本質的な性質」
- 乾: 創始・主導・大胆な拡大
- 坤: 受容・遂行・地道な継続
- 震: 突発・衝撃・急な開始
- 巽: 浸透・適応・徐々の変化
- 坎: リスク・障害・困難への対処
- 離: 判断・可視化・明確化
- 艮: 停止・制約・境界設定
- 兌: 合意・喜び・交流

これは易経の意味論に直結した分類。

### 第2段階: 15卦から選択

本質の八卦が決まったら:
- その八卦が**上卦**の場合: 外部環境・結果として表出
- その八卦が**下卦**の場合: 内部状態・原因として潜在
- もう一方の卦: 対になる要素（補完/対立/影響元）

例: 本質が「震」で、外部要因として現れている場合
→ 震が上卦の8卦から選択
→ 下卦（内部状態）が何かで最終決定

### 第3段階: 爻の判定

64卦が決まってから:
- その卦固有の爻辞を参照可能
- 汎用の「early/mid/late」ではなく、卦の文脈で判定
- 例: 震為雷の初爻「震来虩虩」と六五「震往来厲」は異なる段階

### このアプローチの利点

1. **易経の意味論に直結**: 「本質の八卦」は易経の中心概念
2. **64択→8択→15択**: 段階的絞り込みで判定が容易
3. **爻の文脈依存性を解決**: 64卦が決まってから爻を判定
4. **再現性向上**: 8クラス分類は64クラスより遥かに安定
5. **候補提示との相性**: 15卦をTop-k提示してユーザー選択も可能

### 評価指標案

- 第1段階: 本質八卦のTop-1精度 ≥70%
- 第2段階: 15卦の中でのTop-1精度 ≥60%、Top-3再現率 ≥85%
- 全体: 64卦Top-1 = 第1段階×第2段階 ≥42%

### 懸念点

1. 「本質の八卦」が複数該当する場合の扱い
2. 上卦/下卦の判定（内外モデル）は依然として必要
3. 15卦から1つを選ぶ判定基準の明確化が必要

## Codex (OpenAI GPT-5.2) の批評
**結論（最初に明言）**  
この「本質八卦→15卦選択」案は、現状のまま採用してはならない。理由は単純で、64卦=上卦×下卦という同一構造を“決定順”として言い換えただけで新規性がなく、しかも「本質八卦」を単一に固定するゲートが誤りを確定させて正解を排除する致命的欠陥だからである。

**批判的分析**  
- **1. 論理的整合性**: 「抽象軸の恣意性を除去する」という主張は成立しない。八卦を先に決める時点で解釈の恣意性が温存されるからである。「本質」を「変化・行動の本質的性質」と言い換えているだけで、判定規則が定義されていない（同義反復）。さらに「上卦=外部結果／下卦=内部原因」は易の読法の一部を過剰一般化した誤りである。上下は内外の象として扱えても、原因結果へ一意に落ちない。  
- **2. 実現可能性**: UIとして段階的に絞り込む実装は容易である。しかし“判定できる”ことと“再現性が出る”ことは別である。再現性を出すには「本質八卦」を割り当てるための観測項目・質問票・タイブレーク規則・禁止条件が必須で、ここが最大コストである。現案はそこを空白のまま「8クラス分類は安定」と断言しており、設計が未完成である。  
- **3. 新規性の真偽**: 新規性は低い。64卦が（上卦,下卦）の直積で表現される以上、「先に一方の三爻（八卦）を決めてから相方を決める」は単なる階層化（再パラメータ化）である。15卦という数え上げも組合せの帰結で、新手法ではない。  
- **4. スケーラビリティ**: ハードゲート（本質を1つに固定）方式はスケールすると劣化する。誤った第1段階が入った瞬間に正解候補が候補集合から消滅し、後段がどれだけ賢くても回復不能だからである。規模が増えるほど境界事例が増え、この欠陥が頻発する。  
- **5. 反例・エッジケース**: 「本質」が単一にならない事例が多数存在する（危機対応=坎＋浸透=巽、停止=艮の中で突発=震、可視化=離と合意=兌が同時に主題など）。この設計は二重主題を構造的に表現できず、強制単一選択で誤りを生む。またプロセスが時間で相転移する事例（序盤=震、終盤=艮など）に対し「本質固定」は概念的に破綻する。  
- **6. 代替案との比較**: より優れた代替が存在する。最小の改善は「本質八卦」を捨て、**上卦と下卦を別々に8クラスで推定**して確率分布を保持し、64通りをスコアリングで統合する方式である。これなら誤りが回復不能にならず、曖昧性も扱える。さらに事例DBがあるなら、**類例検索（リトリーバル）＋再ランキング**が再現性と説明可能性を同時に満たす。

**Claudeの意見への反論/修正**  
- 「根本から変更」は誤りである。やっていることは（上卦,下卦）の決定順の入れ替えであり、問題の本体（解釈の規則化）に手を付けていない。  
- 「再現性向上」は根拠がない。再現性は“段階数”で上がらない。“判定規則の明文化”でのみ上がる。現案は規則を提示していない。  
- 提示された精度計算は不正確である。段階精度の単純積を全体Top-1にするのは、条件付き精度の定義が揃っていない限り成立しない。評価設計が粗い。  
- 正しい点もある。15卦に落とす絞り込みはUIとして有効で、64卦決定後に爻判定へ進む順序も妥当である。ただし「順序が妥当」なだけで「判定が可能」にはならない。

**より正確な見解**  
- この問題の本質は「八卦の意味論に直結」ではなく、**観測→判定の写像を検証可能な形で固定すること**である。固定できない限り、段階化は主観を分割しているだけで再現性は上がらない。  
- 採用すべき構造はハードゲートではない。**上卦・下卦それぞれにスコア（確率）を付け、64卦を合成スコアで順位付けしTop-k提示**する方式が正道である。第1段階の誤りで正解が消滅しない設計が必須である。  
- 「本質」を使うなら名称からして誤りである。扱っているのは本質ではなく**支配的特徴（dominant aspect）**であり、複数同時成立を前提にしなければ設計が破綻する。  
- 評価は精度以前に**合意度（アノテータ一致）**を測れ。人間同士で一致しないラベル設計に機械精度を語る資格はない。一致度が確立した後に、Top-k再現率と誤り回復（候補保持）の指標で設計を詰めるべきである。
