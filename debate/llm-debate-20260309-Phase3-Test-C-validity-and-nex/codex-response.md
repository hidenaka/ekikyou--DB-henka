# Codex (OpenAI GPT-5.2) の批評

### 結論（最初に明言）
Test C（Laplacian spectrum の z=-4.74）は**現状のデータ構造（極端な疎グラフ・未カバー多数）から必然的に生じる統計的アーティファクト**であり、**「Q6同型性の実証」には使えない**。Phase 4（予測効用）は**今やる価値がない**。最優先は**測定系（分類器と遷移グラフ定義）の修復と、Test C のヌルモデル再設計**である。

---

### 批判的分析
- **論理的整合性が崩壊している**  
  「同型（Q6）である」を主張するなら、局所構造（=エッジ、Hamming=1 近傍）が一致する必要がある。ところが  
  - Test A（edge overlap）は中立（p=0.61）  
  - Test B（Hamming distance）は反証側（p=0.72, anti）  
  であり、**同型性の必要条件が満たされていない**。この状態で Test C だけを採用して「同型に近い」と言うのは**論理破綻**である。スペクトルは「どのエッジが一致したか」を捨てるため、同型性の主張には弱すぎる。

- **Test C は疎グラフ支配で“勝てる”設計になっている**  
  「17/64ノードしか out-degree>0 がない」時点で、遷移グラフは**大半が孤立・準孤立**である。Laplacian スペクトルは連結成分数・孤立点数・次数分布に強く支配される。つまり Test C が拾っているのは「Q6と似ている構造」ではなく、**“孤立点だらけ”という退化構造の安定性**である。  
  しかも分類器が 34/64 しか出さないなら、再注釈をしても「出現ノード集合」と「孤立点の山」がほぼ固定され、**スペクトルだけが再現性を持つ**。これは“頑健な発見”ではなく**頑健な欠陥**である。

- **p=0.000 は統計報告として誤りである**  
  permutation / bootstrap 系の p 値はゼロにならない。`p=0.000` は丸め誤差では済まない問題で、**検定実装・報告の信頼性に赤信号**が立っている。少なくとも `p < 1/(B+1)` の形で出すべきである。

- **多重検定の扱いが不適切で、勝ち筋だけ拾っている**  
  A〜E の 5 つを回して C だけを「核心」とするのは、補正なしなら**結論の恣意性**そのものになる。しかも他のテストは中立〜反証側が多い。現状の総括「Overall: Weak」が正しい。

- **実現可能性（Phase 4）が成立していない**  
  分類器が 34/64 で、分野によって trigrams=0% が出るなら、予測モデルは「現象」ではなく**ラベリング機構と欠測構造**を予測する。Phase 4 を回しても、出るのは「欠陥の学習結果」であり、研究投資として無駄である。

- **新規性は“主張”だけが新しく、証拠が新しくない**  
  Q6（(Z2)^6）表現自体は一般的な 6-bit 表現であり、新規性は「実データがQ6の力学に従う」という部分にある。そこが現状反証寄りなので、**新規性の核が崩れている**。

- **スケーラビリティは逆に悪化する**  
  11,336件あっても、カバレッジ欠陥がある限りデータを足すほど**偏りが固定**され、未出現クラスは永遠に出ない。規模は武器にならない。

- **反例・エッジケースが既に出ている**  
  「Cuogua symmetry が 2/32 しか valid にならない」時点で、検定対象の定義が現実データに適合していない。これは“弱い結果”ではなく**テスト設計の失敗**である。

---

### Claudeの意見への反論/修正
- 「Test C が唯一頑健だからスペクトルのみ主張してPhase 3を締める」は誤りである。  
  スペクトルが頑健なのは、**同じ欠陥（ノード未カバー・孤立点支配）が頑健**だからである。ここでスペクトルのみを結論にすると、論文査読では「退化構造を同型性と誤認している」と一撃で落ちる。

- 「Phase 4 に進む」は順序が逆である。  
  予測効用は最終局面の評価であり、測定系が壊れている段階でやるのは**指標の自己欺瞞**である。まず Phase 3 の検定が意味を持つ状態（64ノード・分野別トリグラム欠測の解消）に戻す必要がある。

- 「gold set を 500 に拡張」は方向は正しいが甘い。  
  64クラス問題で“500”を雑に増やしても、希少クラスが埋まらず欠測が温存される。必要なのは件数ではなく **カバレッジ制約**である（例：各hexagram最低N件、各trigram×分野で最低N件）。これを満たさない gold set は、Phase 4 どころか Phase 3 の再検証にも使えない。

---

### より正確な見解
- **Test C は「発見」ではなく「検定系の破綻シグナル」**である。現状のスペクトル有意は、同型性の証拠ではなく、孤立点・欠測・次数偏りという**低次統計量**を拾っているだけである。したがって「Q6に近い」と主張してはならない。

- **最も効率の良い次の一手（Phase 3 の再生条件）**  
  1) **遷移グラフの定義を固定**（64ノード上で、どのケースがどの有向辺を生むか、欠測の扱い、閾値で辺を落とすならその根拠）  
  2) **カバレッジを強制する gold set を作る**（「500件」ではなく「各hexagram最低N」「各trigram×分野最低N」を満たすことが条件）  
  3) Test C は **孤立点を除いた誘導部分グラフ**でも同じ効果が出るかを確認し、さらに **次数列（少なくとも入出次数分布）を保存するヌル**で z を出し直す  
  4) その上で Test A/B が改善しないなら、同型性仮説は**棄却**し、研究目的を「I Ching の同型」から「データ上の少数状態遷移の記述」へピボットする

- **Phase 4 のROI判断**  
  現状のまま Phase 4 をやるROIは**負**である。データ欠陥を学習して“当たった気になる”だけで、同型性も予測有用性も証明しない。Phase 4 は、上の 1)〜3) を満たした後に、**ベースライン（頻度・分野・時系列）を厳密に抑えた上で「Q6特徴が上乗せするか」**だけを検証する用途に限定すべきである。
